## 3. 컬러 사용 규칙

### 3-1. 필수 규칙

1. **Primary-400(`#34D399`)은 버튼·아이콘·그래프에만 사용** — 흰 배경 위 텍스트로 직접 사용 금지 (대비율 2.1:1로 WCAG 기준 미달)
2. **텍스트에 Primary 계열 사용 시 Primary-600 이상** 사용 (대비율 4.6:1 이상 확보)
3. **앱 배경은 반드시 `cream-bg(#FFFDF7)`** — 순수 흰색(#FFFFFF) 사용 금지
4. **컨디션 인디케이터 3색은 고정** — 좋음 `success-400`, 보통 `warning-400`, 나쁨 `danger-400`
5. **기록 유형별 카테고리 컬러는 변경 불가** — 사용자 혼란 방지

### 3-2. 대비율 요약 (WCAG 2.1)

| 조합 | 대비율 | 등급 |
|------|--------|------|
| Primary-400 on Primary-900 | 7.2:1 | ✅ AAA |
| Primary-600 on White | 4.6:1 | ✅ AA |
| Primary-400 on White | 2.1:1 | ❌ FAIL — 텍스트 사용 금지 |
| Gray-900 on Cream-BG | 18.4:1 | ✅ AAA |
| Dark Text on Surface | 9.1:1 | ✅ AAA |

### 3-3. 컴포넌트별 컬러 적용 가이드

| 컴포넌트 | 배경 | 텍스트 | 보더/아이콘 |
|----------|------|--------|------------|
| 주요 CTA 버튼 (filled) | `primary-400` | `primary-900` | — |
| 보조 버튼 (outlined) | `cream-bg` | `primary-600` | `primary-400` |
| FAB 버튼 | `primary-400` | — | `white` |
| 헤더 | `primary-900` | `white` | — |
| 카드 | `primary-50` | `gray-900` | `primary-100` |
| 앱 배경 | `cream-bg` | — | — |
| 하단 네비게이션 | `white` | 활성: `primary-900` / 비활성: `gray-500` | — |
| 활성 탭 배경 | `primary-100` | `primary-900` | — |
| 인풋 필드 | `white` | `gray-900` | `gray-200` → 포커스 시 `primary-400` |
| 툴팁 | `gray-800` | `white` | — |
| 응급 버튼 | `danger-400` | `white` | — |
| 컨디션 좋음 뱃지 | `success-50` | `success-900` | `success-400` |
| 컨디션 보통 뱃지 | `warning-50` | `warning-900` | `warning-400` |
| 컨디션 나쁨 뱃지 | `danger-50` | `danger-900` | `danger-400` |
