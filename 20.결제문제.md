
네, **현재 boolean 기반 구조로는 여러 번 결제를 제대로 관리할 수 없습니다**. 문제를 명확히 보겠습니다:

## 현재 구조의 문제

```dart
// purchase_provider.dart
class PurchaseState {
  final bool hasAdditionalPetCredit;  // ⚠️ 반려동물 1회만
  final bool isUnlocked;              // ⚠️ 내보내기 1회만
}
```

**시나리오:**
```
1. 반려동물 추가 결제 (1회차)
   → hasAdditionalPetCredit: true
   
2. 반려동물 추가 (사용)
   → consumeAdditionalPetCredit()
   → hasAdditionalPetCredit: false
   
3. 또 다른 반려동물 추가 결제 (2회차)
   → ? (boolean이므로 다시 true만 될 수 있음)
   
4. 3번과 2번의 결제를 어떻게 구분할 것인가?
```

## 올바른 아키텍처

**구매 이력을 큐/리스트로 관리:**

```dart
class PurchaseTransaction {
  final String transactionId;      // 플랫폼에서 제공하는 고유 ID
  final String productId;          // 'additional_pet_per_use', 'export_data_per_use'
  final DateTime purchasedAt;
  final PurchaseStatus status;     // purchased, consumed, failed
}

class PurchaseState {
  final List<PurchaseTransaction> transactions;  // ✅ 여러 건 추적 가능
}
```

**사용 흐름:**
```
1. A구매 (transactionId: txn_123)
   → transactions: [txn_123(pending)]
   
2. 플랫폼 검증
   → queryPastPurchases()로 txn_123 확인
   
3. A사용 처리
   → transactions: [txn_123(consumed)]
   
4. B구매 (transactionId: txn_456)
   → transactions: [txn_123(consumed), txn_456(pending)]
   
5. 플랫폼 검증
   → queryPastPurchases()로 txn_456 확인
   
6. B사용 처리
   → transactions: [txn_123(consumed), txn_456(consumed)]
```

if (purchaseState.isUnlocked)

## 권장 해결책

```dart
class PurchaseNotifier extends StateNotifier<PurchaseState> {
  // 미사용 구매 찾기
  PurchaseTransaction? _findPendingTransaction(String productId) {
    return state.transactions.firstWhereOrNull(
      (t) => t.productId == productId && t.status == PurchaseStatus.pending,
    );
  }

  // 플랫폼에서 검증 후 실제 사용
  Future<bool> verifyAndConsume(String productId) async {
    final pending = _findPendingTransaction(productId);
    if (pending == null) return false;

    // 플랫폼에 진짜 구매인지 확인
    final purchases = await _service.queryPastPurchases();
    final verified = purchases.any((p) => 
      p.transactionId == pending.transactionId
    );
    
    if (!verified) {
      // 로컬 파일 조작 감지!
      _removeTransaction(pending.transactionId);
      return false;
    }

    // 실제 사용 처리
    _updateTransactionStatus(pending.transactionId, PurchaseStatus.consumed);
    return true;
  }
}
```

이렇게 하면 사용자가 여러 번 구매해도 각각 독립적으로 추적 가능하고, 플랫폼 검증까지 연결됩니다.
