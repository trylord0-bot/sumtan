## 1. 개요

### 1-1. 화면 목적
- 예방접종 / 병원 예약 / 투약 / 식사 / 일일 리마인더 알림을 **단일 통합 리스트**로 표시
- 알림별 **ON/OFF 토글**, **탭 → 수정/재예약**, **스와이프 → 삭제** 제공
- **로컬 푸시 알림** 실제 발송 (`flutter_local_notifications`)
- 단일 알람 발송 후 **완료 처리 / 재예약 / 숨김** 플로우 지원

### 1-2. 라우트
```dart
GoRoute(path: '/notifications', builder: (_, __) => const AlarmScreen())
```

### 1-3. 패키지 의존성
```yaml
dependencies:
  flutter_local_notifications: ^18.x
  timezone: ^0.9.x
```

## 3. 데이터 모델

### 3-1. `alarms` 테이블 스키마
```sql
CREATE TABLE alarms (
  id            INTEGER PRIMARY KEY AUTOINCREMENT,
  pet_id        INTEGER NOT NULL,
  type          TEXT NOT NULL,
  label         TEXT,
  scheduled_at  TEXT,
  repeat_rule   TEXT DEFAULT 'none',
  repeat_time   TEXT,
  is_enabled    INTEGER DEFAULT 1,
  is_done       INTEGER DEFAULT 0,   -- 완료 처리 여부 (v3 추가)
  done_at       TEXT,                -- 완료 처리 일시 ISO8601 (v3 추가)
  record_id     INTEGER,
  created_at    TEXT NOT NULL
)
```

### 3-2. `type` 값 정의
| 값 | 이모지 | 설명 |
|----|--------|------|
| `vaccination` | 💉 | 예방접종 D-day 알림 |
| `hospital` | 🏥 | 병원 예약 알림 |
| `medication` | 💊 | 투약 리마인더 |
| `meal` | 🍽️ | 식사 시간 알림 |
| `daily` | 📋 | 일일 기록 리마인더 |

### 3-3. `repeat_rule` 값 정의
| 값 | 설명 |
|----|------|
| `none` | 1회성 (`scheduled_at` 기준) |
| `daily` | 매일 반복 (`repeat_time` 기준) |
| `weekday` | 평일만 반복 |
| `weekend` | 주말만 반복 |
| `weekly` | 매주 반복 |
| `monthly` | 매달 반복 |

### 3-4. 알람 상태 정의 (v3 추가)
```dart
enum AlarmStatus {
  upcoming,   // 예정 (scheduled_at > 오늘)
  todayPending, // 오늘이지만 미완료 (scheduled_at == 오늘 && is_done == 0)
  past,       // 날짜 경과 + 미완료 (scheduled_at < 오늘 && is_done == 0)
  done,       // 완료 처리됨 (is_done == 1)
  repeat,     // 반복 알림 (repeat_rule != 'none')
}

extension AlarmStatusX on Alarm {
  AlarmStatus get status {
    if (repeatRule != 'none') return AlarmStatus.repeat;
    if (isDone) return AlarmStatus.done;
    final scheduled = DateTime.parse(scheduledAt!);
    final today = DateTime.now();
    if (isSameDay(scheduled, today) && !isDone) return AlarmStatus.todayPending;
    if (scheduled.isBefore(today)) return AlarmStatus.past;
    return AlarmStatus.upcoming;
  }
}
```


## 5. NotificationService

### 5-1. 채널 정의 (Android)
| 채널 ID | 채널명 | 알림 타입 |
|---------|--------|----------|
| `meal_reminder` | 식사 알림 | `meal` |
| `medication_reminder` | 투약 알림 | `medication` |
| `health_reminder` | 건강 알림 | `vaccination`, `hospital` |
| `daily_reminder` | 일일 리마인더 | `daily` |

### 5-2. 푸시 ID 전략
- `alarms.id` (DB PK) = `notification id` **1:1 매핑**
- 수정/재예약 시 동일 id로 `zonedSchedule` 덮어쓰기
- 삭제 / 토글 OFF / 완료 처리 시 `cancel(alarm.id!)` 호출

### 5-3. 주요 메서드
```dart
class NotificationService {
  Future<void> init();
  Future<bool> requestPermission();
  Future<void> schedule(Alarm alarm);
  Future<void> cancel(int id);
  Future<void> cancelAll();
  Future<void> rescheduleAll(List<Alarm> alarms);
}
```

### 5-4. 푸시 알림 액션 버튼 (v3 추가)
```dart
// flutter_local_notifications 액션 버튼 설정
const AndroidNotificationAction doneAction = AndroidNotificationAction(
  'action_done',
  '✅ 완료했어요',
  showsUserInterface: true,
);
const AndroidNotificationAction laterAction = AndroidNotificationAction(
  'action_later',
  '나중에',
  showsUserInterface: false,
  cancelNotification: true,
);

// 액션 응답 처리
void onDidReceiveNotificationResponse(NotificationResponse response) {
  final alarmId = int.parse(response.payload!);
  switch (response.actionId) {
    case 'action_done':
      // 앱 실행 → 완료 확인 다이얼로그 표시
      context.go('/notifications');

      break;
    case 'action_later':
      // 푸시만 닫힘, 알람은 todayPending 상태로 리스트에 유지
      break;
    default:
      // 푸시 본문 탭 → 알림 화면 이동
      context.go('/notifications');
  }
}
```

### 10-3. markDone 처리 내용
```dart
Future<void> markDone(int alarmId) async {
  // 1. alarms 테이블 업데이트
  await _repo.update(alarm.copyWith(isDone: true, doneAt: DateTime.now().toIso8601String()));

  // 2. 푸시 알림 취소
  await _notificationService.cancel(alarmId);

  // 3. records 테이블에 자동 기록 생성
  await _recordRepo.insert(Record(
    petId: alarm.petId,
    type: alarm.type,
    label: alarm.label,
    recordedAt: DateTime.now().toIso8601String(),
    sourceAlarmId: alarmId,
  ));

  // 4. 상태 갱신
  state = AsyncValue.data([...alarms]);
}
```

### 12-3. reschedule 처리
```dart
Future<void> reschedule(Alarm alarm) async {
  // 1. is_done 리셋 + scheduled_at 업데이트
  final updated = alarm.copyWith(isDone: false, doneAt: null);
  await _repo.update(updated);

  // 2. 기존 푸시 취소 후 재스케줄
  await _notificationService.cancel(alarm.id!);
  await _notificationService.schedule(updated);

  state = AsyncValue.data([...alarms]);
}
```

## 18. 구현 시 주의사항

1. **FAB 위치** — 탭바 중앙 버튼만 사용. `FloatingActionButton`(우하단) 추가 금지.
2. **타입 선택 → 폼 전환** — "다음" 버튼 없이 타입 아이템 탭 즉시 Step 2 바텀시트 오픈.
3. **수정 폼 진입** — `AlarmListItem` 탭 → `AlarmFormSheet(editAlarm: alarm)`.
4. **재예약 폼 진입** — `past` 상태 아이템 탭 → `AlarmFormSheet(editAlarm: alarm, isReschedule: true)`.
5. **스와이프 삭제** — `Dismissible(direction: endToStart)`, 빨간 배경 + 🗑️.
6. **삭제 버튼** — 수정/재예약 폼 하단에만 노출. 추가 폼에는 표시 안 함.
7. **완료 처리** — `markDone()` 호출 시 records 테이블 자동 기록 생성 필수.
8. **todayPending 탭 무시** — 완료/나중에 버튼으로만 조작. 카드 탭 이벤트 차단.
9. **done 상태 탭 무시** — 완료된 아이템은 탭 비활성화.
10. **지난 알림 숨김 상태** — `pastAlarmsVisibleProvider`로 전역 관리. 앱 재시작 시 기본 접힘.
11. **iOS 권한 요청** — `requestPermission()` 첫 진입 시 호출.
12. **Android 13+** — `POST_NOTIFICATIONS` 런타임 권한 요청 필수.
13. **timezone 초기화** — `tz.initializeTimeZones()` + `tz.setLocalLocation()` main에서 호출.
14. **앱 재시작 알림 재등록** — `rescheduleAll()` 앱 초기화 시 호출.
15. **alarms.id ↔ notification id 1:1 매핑** — 별도 id 관리 불필요.
16. **다중 펫** — `alarm.petId` 기준 필터링, 알림 문구에 펫 이름 포함.
17. **섹션 구분** — `repeat_rule == 'none'`이면 "예정된 알림", 그 외 "반복 알림" 섹션.
18. **토글 이벤트 버블링** — 토글 탭과 카드 탭 충돌 방지: `GestureDetector(behavior: HitTestBehavior.opaque)`.
