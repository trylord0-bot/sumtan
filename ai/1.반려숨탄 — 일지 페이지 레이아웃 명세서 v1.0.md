## 2. 전체 레이아웃 구조

```
┌─────────────────────────────┐  ← 상태바 (44dp, primary-50 배경)
├─────────────────────────────┤  ← 헤더 (56dp, primary-50 배경)
│  [일지]   [🐶 콩이 ▾]  [☰] │
├─────────────────────────────┤
│                             │
│     ◀  2026년 5월  ▶        │  ← 월 캘린더 (고정, flex-shrink:0)
│  일 월 화 수 목 금 토        │
│  [날짜 셀 × 최대 6주]        │
│  (각 셀에 카테고리 컬러 도트) │
│                             │
├─────────────────────────────┤  ← 구분선
│  5월 기록 (12건)             │
│  ┌─────────────────────┐   │
│  │ 날짜 그룹 헤더       │   │  ← 이벤트 리스트
│  │ [카드] [카드] ...    │   │     (독립 스크롤, flex:1)
│  └─────────────────────┘   │
│  ...                        │
├─────────────────────────────┤
│  [🏠][📔][📅][📊][⚙️]      │  ← 하단 탭바 (64dp, 고정)
└─────────────────────────────┘
      ↑ FAB (우하단, 탭바 위 76dp)
```

### 2-1. 레이아웃 핵심 규칙

| 영역 | flex 속성 | 스크롤 |
|------|-----------|--------|
| 상태바 | `flex-shrink: 0` | 없음 (고정) |
| 헤더 | `flex-shrink: 0` | 없음 (고정) |
| **월 캘린더** | **`flex-shrink: 0`** | **없음 (고정)** |
| **이벤트 리스트** | **`flex: 1`** | **독립 `overflow-y: auto`** |
| 탭바 | `position: absolute; bottom: 0` | 없음 (고정) |

> ⚠️ 날짜 클릭 시 캘린더는 절대 스크롤되지 않으며, 이벤트 리스트 영역만 해당 날짜 그룹으로 스크롤된다.

## 3. 월 캘린더 섹션

### 3-1. 월 헤더

| 요소 | 스펙 |
|------|------|
| 배경 | `white` |
| 패딩 | `space-4` (16dp) 상하, `space-4` (14dp) 좌우 |
| 이전/다음 월 버튼 | 30×30dp, `radius-full`, 배경 `primary-50`, 아이콘 `‹` `›` |
| 월 타이틀 | `type-body-lg` (16sp · W700), 색상 `primary-900` |
| 요일 헤더 | 7열 그리드, 10sp · W600, 색상 `gray-400` |
| 일요일 색상 | `#EF4444` (red) |
| 토요일 색상 | `#3B82F6` (blue) |

### 3-2. 날짜 셀

| 상태 | 스펙 |
|------|------|
| 기본 | 배경 없음, 숫자 `gray-700`, 12sp · W500 |
| 오늘 | 숫자 배경 `primary-400` 원형, 텍스트 `white` · W700 |
| 선택됨 (오늘 아님) | 숫자 배경 `primary-900` 원형, 텍스트 `white` · W700 |
| 다른 달 | 숫자 색상 `gray-300` |
| 호버 | 셀 배경 `primary-50` |
| 최소 높이 | 48dp |

### 3-3. 이벤트 도트

- 날짜 셀 숫자 아래에 카테고리별 컬러 도트 표시
- 같은 날짜에 같은 카테고리 이벤트가 여러 개여도 **도트는 카테고리당 1개**
- 도트 크기: 4×4dp, `radius-full`
- **개수 제한 없음** — `flex-wrap` 적용으로 2줄 이상도 모두 표시
- 도트 간격: `gap: 2dp`, `row-gap: 2dp`

| 카테고리 | 도트 색상 |
|----------|----------|
| 식사 (meal) | `#F59E0B` (amber) |
| 산책 (walk) | `#3B82F6` (blue) |
| 투약 (health) | `#EC4899` (pink) |
| 병원 (hospital) | `#EF4444` (red) |
| 체중 (weight) | `#10B981` (green) |
| 메모 (memo) | `#6B7280` (gray) |

### 3-4. 날짜 클릭 동작

1. 선택된 날짜 셀 → `selected` 상태로 스타일 변경
2. 캘린더는 **움직이지 않음** (고정 유지)
3. 이벤트 리스트 스크롤 영역에서 해당 날짜 그룹 요소로 `scrollTo({ behavior: 'smooth' })`
4. 해당 날짜 기록이 없으면 리스트 최상단으로 이동

## 4. 이벤트 리스트 섹션

### 4-1. 섹션 헤더

| 요소 | 스펙 |
|------|------|
| 텍스트 | `"N월 기록 (N건)"` |
| 스타일 | 13sp · W700, 색상 `gray-500` |
| 표시 범위 | 현재 보고 있는 **월 전체** 이벤트 |

### 4-2. 날짜 그룹

- 날짜별로 이벤트를 묶어 그룹 헤더 + 카드 목록으로 표시
- 날짜 내림차순 정렬 (최신 날짜가 위)
- 같은 날짜 내 이벤트는 시간 오름차순 정렬
- 각 그룹에 `id="group-{YYYY-MM-DD}"` 부여 (스크롤 타겟)

**날짜 그룹 헤더 배지**

| 상태 | 배경 | 텍스트 |
|------|------|--------|
| 일반 | `primary-100` | `primary-900` |
| 오늘 | `primary-400` | `white` |
| 선택됨 | `primary-900` | `white` |

### 4-3. 이벤트 카드

| 요소 | 스펙 |
|------|------|
| 배경 | `white` |
| 보더 레디우스 | `radius-lg` (16dp) |
| 그림자 | `0 1px 4px rgba(0,0,0,.06)` |
| 패딩 | 12dp 상하, 14dp 좌우 |
| 레이아웃 | 가로 flex: [카테고리 아이콘] + [본문] + [시간] |
| 카테고리 아이콘 | 38×38dp, `radius-md`, 카테고리 배경색 |
| 제목 | 14sp · W700, `gray-900` |
| 설명 | 12sp, `gray-500`, 1줄 말줄임 |
| 시간 | 11sp, `gray-400`, 우측 정렬 |
| 미디어 썸네일 스트립 | 카드 하단, 40×40dp 썸네일 가로 나열 |
| 클릭 동작 | 이벤트 상세 바텀 시트 오픈 |

## 7. 이벤트 추가 플로우 (바텀 시트 2단계)

### 7-1. Step 1 — 카테고리 선택 시트

| 항목 | 스펙 |
|------|------|
| 트리거 | FAB `＋` 클릭 |
| 시트 높이 | 콘텐츠 높이에 맞춤 (max-height: 90%) |
| 배경 오버레이 | `rgba(0,0,0,.45)` |
| 보더 레디우스 | 상단 `radius-xl` (24dp) |
| 핸들 바 | 36×4dp, `gray-300`, 상단 중앙 |
| 레이아웃 | 3열 그리드 (식사·산책·투약 / 병원·체중·메모) |
| 카테고리 버튼 | 배경 `gray-100`, 선택 시 테두리 `primary-400` + 배경 `primary-50` |
| 다음 버튼 | Filled Primary, 카테고리 미선택 시 `disabled` |
| 닫기 | 우상단 ✕ 버튼 또는 오버레이 탭 |

### 7-2. Step 2 — 내용 입력 시트

| 항목 | 스펙 |
|------|------|
| 트리거 | Step 1에서 카테고리 선택 후 "다음 →" 클릭 |
| 헤더 | 좌측 `‹` 뒤로가기 버튼 + 카테고리명 타이틀 + 우측 ✕ |
| 공통 필드 | 날짜(date picker) + 시간(time picker), 제목(text input) |
| 카테고리별 필드 | 5절 카테고리 표 참조 |
| 메모 필드 | textarea, 72dp 높이 |

#### 7-2-1. 미디어 첨부 영역

| 항목 | 스펙 |
|------|------|
| 위치 | 메모 필드 아래, 저장 버튼 위 |

| 레이블 | "📎 사진 · 영상 첨부" |
| 첨부 버튼 3종 | **사진 촬영** (📷) / **앨범에서 선택** (🖼️) / **동영상** (🎬) |

## 10. 인터랙션 흐름 요약

```
[캘린더 날짜 클릭]
    → 캘린더 고정 유지
    → 이벤트 리스트에서 해당 날짜 그룹으로 smooth scroll

[FAB ＋ 클릭]
    → Step1: 카테고리 선택 시트
        → 카테고리 선택 후 "다음 →"
            → Step2: 내용 입력 시트
                → 날짜/시간/제목/카테고리별 필드/메모 입력
                → 사진 촬영 / 앨범 선택 / 동영상 첨부
                → 첨부 파일 썸네일 프리뷰 (✕로 개별 제거 가능)
                → "저장하기" → 시트 닫기 → 캘린더·리스트 갱신

[이벤트 카드 클릭]
    → 상세 바텀 시트
        → 카테고리별 상세 필드 표시
        → 첨부 미디어 3열 그리드 표시
            → 사진: 썸네일
            → 동영상: 썸네일 + ▶️ 재생 오버레이 (클릭 시 플레이어)
        → [✏️ 수정] → Step2 폼 재오픈 (기존 값 유지)
        → [🗑️ 삭제] → 삭제 확인 다이얼로그 → 확인 시 삭제
```
