## 1. 예방접종 / 병원 카테고리 — 시간 필수 입력

### 배경
현재 `vaccination`(예방접종), `hospital`(병원) 카테고리는 날짜만 필수이고 시간은 선택 사항이다.
시간이 없을 경우 `_save()`에서 오전 9시로 강제 저장되는 문제가 있다.

### 수정 내용
- **파일:** `lib/features/alarm/presentation/widgets/alarm_form_sheet.dart`
- `_validate()` 메서드에서 `vaccination`, `hospital` 케이스에 시간 필수 검증 추가
  - `vaccination`: 예정 시각 미선택 시 → `'예정 시각을 선택해주세요'` 반환
  - `hospital`: 예약 시각 미선택 시 → `'예약 시각을 선택해주세요'` 반환
- `_save()` 메서드에서 `vaccination`, `hospital` 케이스의 시간 폴백(`?? 9`, `?? 0`)을 제거하고 `!` 단언으로 변경
  - 단, `medication`의 `repeatRule == 'none'` 케이스 등 시간이 선택 사항인 케이스는 폴백 유지

### 검증 조건 요약
| 카테고리 | 날짜 필수 | 시간 필수 |
|----------|-----------|-----------|
| `vaccination` | ✅ | ✅ (변경) |
| `hospital` | ✅ | ✅ (변경) |
| `medication` | 규칙에 따라 다름 | 규칙에 따라 다름 |
| `meal` | 현행 유지 | 현행 유지 |
| `daily` | 현행 유지 | 현행 유지 |

## 2. 알림 리스트 정렬 기준 추가

### 배경
현재 각 섹션의 알림 항목은 DB 삽입 순서(등록 순서)로 표시된다.
사용자가 직관적으로 일정을 파악할 수 있도록 섹션별 정렬 기준을 추가한다.

### 수정 내용
- **파일:** `lib/features/alarm/presentation/alarm_screen.dart`
- 각 섹션 리스트를 `where().toList()` 후 `.sort()` 로 정렬 적용

#### 섹션별 정렬 기준

| 섹션 | 변수명 | 정렬 기준 |
|------|--------|----------|
| 오늘 예정 | `todayPending` | `scheduledAt` **오름차순** (이른 시간 먼저) |
| 예정된 알림 | `upcoming` | `scheduledAt` **오름차순** (가까운 날짜 먼저, `null`은 맨 뒤) |
| 지난 알림 / 완료 | `pastOrDone` | `scheduledAt` **내림차순** (최근 날짜 먼저, `null`은 맨 뒤) |
| 반복 알림 | `repeat` | `repeatTime` **오름차순** (이른 시간 먼저, `null`은 맨 뒤) |

#### 정렬 적용 예시 (참고용)
```dart
// todayPending — scheduledAt 오름차순
todayPending.sort((a, b) {
  final at = a.scheduledAt != null ? DateTime.parse(a.scheduledAt!) : DateTime(9999);
  final bt = b.scheduledAt != null ? DateTime.parse(b.scheduledAt!) : DateTime(9999);
  return at.compareTo(bt);
});

// upcoming — scheduledAt 오름차순 (null 맨 뒤)
upcoming.sort((a, b) {
  final at = a.scheduledAt != null ? DateTime.parse(a.scheduledAt!) : DateTime(9999);
  final bt = b.scheduledAt != null ? DateTime.parse(b.scheduledAt!) : DateTime(9999);
  return at.compareTo(bt);
});

// pastOrDone — scheduledAt 내림차순 (null 맨 뒤)
pastOrDone.sort((a, b) {
  final at = a.scheduledAt != null ? DateTime.parse(a.scheduledAt!) : DateTime(0);
  final bt = b.scheduledAt != null ? DateTime.parse(b.scheduledAt!) : DateTime(0);
  return bt.compareTo(at);
});

// repeat — repeatTime 오름차순 (null 맨 뒤)
// repeatTime 형식: "HH:mm" 문자열 → 문자열 비교만으로 정렬 가능
repeat.sort((a, b) {
  final at = a.repeatTime ?? '99:99';
  final bt = b.repeatTime ?? '99:99';
  return at.compareTo(bt);
});
```

### 참고 사항
- `repeat` 알림은 `AlarmStatus.repeat`으로 이미 별도 분리되어 있으므로 `todayPending`, `upcoming` 섹션에 혼입되지 않는다. 별도 필터링 작업 불필요.
- `past_alarms_section.dart`는 `alarm_screen.dart`에서 이미 정렬된 리스트를 전달받으므로 별도 수정 불필요.
- `repeatTime`은 `"HH:mm"` 형식의 문자열이므로 문자열 직접 비교로 시간 순 정렬이 가능하다.
