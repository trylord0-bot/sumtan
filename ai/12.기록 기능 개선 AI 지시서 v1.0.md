# 반려숨탄 — 기록 기능 개선 AI 지시서 v1.0

> 본 문서는 반려동물 건강관리 앱 **반려숨탄**의 기록 기능 개선 사항을 AI 개발자에게 지시하기 위한 명세서입니다.
> Flutter + Clean Architecture + flutter_riverpod 기반 프로젝트에 적용합니다.

## 2. 접종 기록 개선

### 2-1. 종별 백신 목록 분리

반려동물 종(강아지/고양이)에 따라 선택 가능한 백신 목록을 분리한다.

**강아지 🐶**
- 종합백신 (DHPPL)
- 코로나장염
- 켄넬코프
- 광견병
- 인플루엔자
- 기타

**고양이 🐱**
- 종합백신 (FVRCP)
- 광견병
- 백혈병 (FeLV)
- 클라미디아
- 기타

> **용어 변경:** `코로나` → `코로나장염` (사람 코로나와의 혼동 방지)

## 3. 빗질 기록 추가

### 적용 대상
- **강아지 🐶 + 고양이 🐱 공용** (단일 폼)
- 기존 고양이에만 존재하던 빗질 카테고리를 강아지에도 추가
- 폼은 공용 하나로 통합 (`brushing_form.dart` 신규 생성)

### 최종 필드 구성
| # | 필드 | 타입 | 비고 |
|---|------|------|------|
| 1 | 📅 날짜/시간 | DateTime | 필수 |
| 2 | 🪮 빗질 부위 | 다중 선택 칩 | 전체 / 등 / 배 / 꼬리 / 얼굴 / 발 |
| 3 | ⏱️ 소요 시간 | 숫자 입력 (분) | 선택 |
| 4 | 📝 메모 | 텍스트 | 선택 |
| 5 | 📎 사진/동영상 | 기존 첨부 기능 재사용 | 선택 |

### 구현 시 수정 파일
1. `brushing_form.dart` — 신규 생성 (강아지/고양이 공용)
2. 카테고리 시트 — 강아지/고양이 모두 `brushing` 카테고리 추가
3. 라우팅 — `brushing_form` 연결
4. **홈 화면** — `brushing` 카테고리 아이콘/타일 표시 추가 (기존 카테고리 표시 패턴 동일하게 적용)
5. **일지 리스트 화면** — `brushing` 기록 항목이 날짜별 리스트에 표시되도록 추가 (기존 기록 아이템 UI 패턴 동일하게 적용)

## 공통 지침

- 사진/동영상 첨부는 **기존 프로젝트의 공용 첨부 기능**을 재사용할 것
- 날짜/시간 선택은 **기존 프로젝트의 공용 DateTimePicker**를 재사용할 것
- 모든 폼은 **기존 폼의 UI/UX 패턴**을 동일하게 따를 것
- Clean Architecture 레이어 구조 준수 (domain / data / presentation)
- 상태 관리는 `flutter_riverpod` 사용
