# 🛠️ AI 지시서: `url_launcher` 외부 앱 호출 오류 전체 수정

> **프로젝트**: `sumtan_app`
> **작성일**: 2026-05-08
> **대상 파일**: `AndroidManifest.xml`, `Info.plist`, `settings_screen.dart`

---

## 📋 개요

설정 화면(`settings_screen.dart`)의 **피드백 보내기** 기능과 햄버거 메뉴의 **주변 병원 찾기** 기능이 Android/iOS 양쪽에서 외부 앱(브라우저, 이메일)을 정상적으로 호출하지 못하는 문제를 수정한다.

---

## 🚨 문제 현황

| 기능 | 플랫폼 | 증상 | 원인 |
|------|--------|------|------|
| 주변 병원 찾기 (http/https) | Android | 브라우저가 열리지 않음 | `<queries>`에 http/https scheme 미선언 |
| 주변 병원 찾기 (http/https) | iOS | 브라우저가 열리지 않음 | `LSApplicationQueriesSchemes` 미선언 |
| 피드백 보내기 (mailto) | Android | "이메일 앱을 찾을 수 없어요" 토스트 출력 | `<queries>`에 mailto scheme 미선언 |
| 피드백 보내기 (mailto) | iOS | "이메일 앱을 찾을 수 없어요" 토스트 출력 | `LSApplicationQueriesSchemes` 미선언 |

---

## 🔧 수정 지시

### 1. `android/app/src/main/AndroidManifest.xml`

`<manifest>` 태그 바로 아래(= `<application>` 태그 위)에 아래 `<queries>` 블록을 추가한다.
기존에 `<queries>` 블록이 있다면 내용을 병합한다.

```xml
<queries>
    <!-- 주변 병원 찾기: 외부 브라우저 호출 -->
    <intent>
        <action android:name="android.intent.action.VIEW" />
        <data android:scheme="https" />
    </intent>
    <intent>
        <action android:name="android.intent.action.VIEW" />
        <data android:scheme="http" />
    </intent>

    <!-- 피드백 보내기: 이메일 앱 호출 -->
    <intent>
        <action android:name="android.intent.action.SENDTO" />
        <data android:scheme="mailto" />
    </intent>
</queries>
```

---

### 2. `ios/Runner/Info.plist`

`</dict>` 바로 위에 아래 블록을 추가한다.
현재 `Info.plist`에는 `LSApplicationQueriesSchemes` 키가 **전혀 없으므로** 신규 추가한다.

```xml
<!-- 외부 앱 호출 허용 (url_launcher 필수 설정) -->
<key>LSApplicationQueriesSchemes</key>
<array>
    <string>https</string>
    <string>http</string>
    <string>mailto</string>
</array>
```

---

### 3. `lib/features/settings/presentation/settings_screen.dart`

#### 현재 코드 (수정 전)

아래 코드를 찾는다:

```dart
onTap: () async {
  final uri = Uri.parse(
    'mailto:feedback@sumtan.app?subject=%EB%B0%98%EB%A0%A4%EC%88%A8%ED%83%84%20%ED%94%BC%EB%93%9C%EB%B0%B1',
  );
  if (await canLaunchUrl(uri)) {
    await launchUrl(uri);
  } else {
    if (context.mounted) {
      showTopToast(context, '이메일 앱을 찾을 수 없어요');
    }
  }
},
```

#### 수정 후 코드

위 코드를 아래로 교체한다:

```dart
onTap: () async {
  final Uri emailUri = Uri(
    scheme: 'mailto',
    path: 'feedback@sumtan.app',
    queryParameters: {
      'subject': '반려숨탄 피드백',
    },
  );
  try {
    final launched = await launchUrl(
      emailUri,
      mode: LaunchMode.externalApplication,
    );
    if (!launched && context.mounted) {
      showTopToast(context, '이메일 앱을 찾을 수 없어요');
    }
  } catch (e) {
    if (context.mounted) {
      showTopToast(context, '이메일 앱을 찾을 수 없어요');
    }
  }
},
```

#### 변경 이유

| 항목 | 변경 전 | 변경 후 | 이유 |
|------|---------|---------|------|
| URL 생성 방식 | `Uri.parse('mailto:...?subject=인코딩된문자열')` | `Uri(scheme: 'mailto', queryParameters: {...})` | 한글 subject/body 자동 인코딩 처리 |
| 실행 방식 | `canLaunchUrl()` 체크 후 `launchUrl()` | `launchUrl()` 직접 호출 후 반환값 확인 | Android 11+에서 `canLaunchUrl()`이 Manifest 미설정 시 항상 `false` 반환 |
| LaunchMode | 미지정 (기본값) | `LaunchMode.externalApplication` | 외부 이메일 앱 강제 호출 보장 |
