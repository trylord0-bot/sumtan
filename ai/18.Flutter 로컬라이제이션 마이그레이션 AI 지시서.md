# Flutter 로컬라이제이션 마이그레이션 AI 지시서

> **프로젝트:** `sumtan_app`  
> **목표:** 커스텀 `app_localizations.dart` 단일 파일 방식 → Flutter 공식 ARB 방식으로 전환  
> **대상 파일 경로:** `sumtan_app/` 기준으로 작성

---

## 📋 현재 상태 요약

| 항목 | 현재 상태 |
|------|-----------|
| 번역 방식 | `lib/app/localization/app_localizations.dart` 단일 파일 (107KB) |
| 번역 호출 방식 | `context.lt('한글 원문')` — 한글 원문이 키 역할 |
| 하드코딩 위치 | `health_guide_screen.dart` 등 일부 `title:` 파라미터에 한글 직접 입력 |

## 🎯 Phase 1 — 프로젝트 설정 변경

### 작업 1-1. `pubspec.yaml` 수정

아래 항목을 추가/확인하라:

```yaml
dependencies:
  flutter:
    sdk: flutter
  flutter_localizations:   # 이미 있으면 유지
    sdk: flutter

flutter:
  generate: true           # 반드시 추가
```

`sumtan_app/l10n.yaml` 파일을 새로 생성하고 아래 내용을 입력하라:

```yaml
arb-dir: lib/app/l10n
template-arb-file: app_en.arb
output-localization-file: app_localizations.dart
output-dir: lib/app/l10n/generated
nullable-getter: false
```

## 🎯 Phase 5 — Extension 및 Provider 수정

### 작업 5-1. `BuildContext` Extension 교체

기존 `app_localizations.dart`에 있던 extension을 삭제하고,  
`lib/app/l10n/l10n_extension.dart` 파일을 새로 생성하라:

```dart
import 'package:flutter/material.dart';
import 'generated/app_localizations.dart';

extension AppLocalizationsX on BuildContext {
  AppLocalizations get l10n => AppLocalizations.of(this);
}
```

### 작업 5-2. `LocaleController` 이전

기존 `app_localizations.dart`의 `LocaleController`와 `localeControllerProvider`를  
`lib/app/l10n/locale_controller.dart`로 분리하여 이전하라:

```dart
import 'package:flutter/material.dart';
import 'package:flutter_riverpod/flutter_riverpod.dart';
import 'package:shared_preferences/shared_preferences.dart';
import 'generated/app_localizations.dart';

final localeControllerProvider =
    AsyncNotifierProvider<LocaleController, Locale?>(LocaleController.new);

class LocaleController extends AsyncNotifier<Locale?> {
  static const _storageKey = 'app_locale';

  @override
  Future<Locale?> build() async {
    final prefs = await SharedPreferences.getInstance();
    final tag = prefs.getString(_storageKey);
    if (tag == null || tag == 'system') return null;
    // AppLocalizations.supportedLocales에서 매칭되는 Locale 반환
    return AppLocalizations.supportedLocales.firstWhere(
      (l) => l.toLanguageTag() == tag,
      orElse: () => const Locale('en'),
    );
  }

  Future<void> setLocale(Locale? locale) async {
    final prefs = await SharedPreferences.getInstance();
    if (locale == null) {
      await prefs.setString(_storageKey, 'system');
    } else {
      await prefs.setString(_storageKey, locale.toLanguageTag());
    }
    state = AsyncData(locale);
  }
}
```

## 🎯 Phase 7 — 전체 스캔 및 검증

### 작업 7-1. 한글 하드코딩 전수 조사

터미널에서 아래 명령어를 실행하여 남아있는 한글 문자열을 모두 찾아라:

```bash
grep -rn "[가-힣]" lib/ --include="*.dart" \
  --exclude-dir="generated" \
  --exclude="app_localizations.dart"
```

출력된 모든 항목을 ARB 키로 등록하고 `context.l10n.키명`으로 교체하라.

### 작업 7-2. `context.lt(` 잔존 여부 확인

```bash
grep -rn "context\.lt(" lib/ --include="*.dart"
grep -rn "\.t('" lib/ --include="*.dart"
grep -rn "l10n\.text(" lib/ --include="*.dart"
```

위 명령어 결과가 **0건**이 될 때까지 교체를 반복하라.

### 작업 7-3. ARB 키 누락 검증

```bash
flutter gen-l10n
```

빌드 에러가 발생하면 에러 메시지에 표시된 누락 키를 모든 ARB 파일에 추가하라.

### 작업 7-4. 앱 실행 및 언어별 확인

```bash
# 각 언어로 실행하여 번역 확인
flutter run --dart-define=FORCE_LOCALE=ko
flutter run --dart-define=FORCE_LOCALE=en
flutter run --dart-define=FORCE_LOCALE=ja
```

또는 `app.dart`에서 임시로 locale을 고정하여 확인하라:

```dart
// 테스트용 — 확인 후 반드시 제거

```

모든 화면에서 새 방식으로 교체가 완료되면:

1. `lib/app/localization/app_localizations.dart` 파일을 **삭제**하라.
2. 해당 파일을 import하는 모든 코드를 새 경로로 교체하라:
   ```dart
   // ❌ 삭제
   import '../../../app/localization/app_localizations.dart';
   
   // ✅ 교체
   import '../../../app/l10n/generated/app_localizations.dart';
   import '../../../app/l10n/l10n_extension.dart';
   ```
3. `flutter pub get` 후 `flutter gen-l10n` 재실행하라.
4. `flutter analyze`를 실행하여 에러가 없는지 확인하라.

## ⚠️ 주의사항

1. **ARB 키 네이밍 규칙:** `camelCase` 사용, 특수문자/이모지 포함 금지  
   예) `tabDog` ✅ / `tab_dog` ❌ / `🐶탭` ❌

2. **`generated/` 폴더는 `.gitignore`에 추가하지 말 것** — CI/CD에서 `flutter gen-l10n`을 실행하거나, 팀원 모두가 로컬에서 생성해야 하므로 커밋에 포함하는 것을 권장함

3. **`health_guide_screen.dart`의 `_item()` 함수** 내부에 한글 문자열이 대량으로 포함되어 있음.  
   이 함수들은 의료 정보 텍스트이므로 번역 품질이 중요함.  
   자동 번역보다 **전문 번역가 검수**를 권장함.

4. **마이그레이션 중 앱이 깨지지 않도록** Phase별로 브랜치를 분리하여 작업하라:
   ```
   git checkout -b feature/l10n-migration-phase1
   ```
