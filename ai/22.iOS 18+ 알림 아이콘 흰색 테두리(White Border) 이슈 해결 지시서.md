# iOS 18+ 알림 아이콘 흰색 테두리(White Border) 이슈 해결 지시서

## 📌 컨텍스트 (Context)
현재 Flutter 프로젝트의 iOS 빌드에서, iOS 18 이상의 기기로 로컬/리모트 알림(Notification) 수신 시 앱 아이콘 테두리에 원치 않는 얇은 흰색 선(White Border/Container)이 강제로 렌더링되는 이슈가 발생하고 있습니다. 

이는 iOS 18부터 도입된 다크 모드(Dark Mode) 및 틴트 모드(Tinted) 아이콘 시스템의 렌더링 방식 변경 때문입니다. 기존의 단일 꽉 찬(Solid) 이미지만 제공할 경우, 시스템이 대비를 맞추기 위해 강제로 테두리를 추가합니다. 이를 해결하기 위해 안드로이드의 Adaptive Icon과 유사하게 전경(Foreground)과 배경(Background)을 분리하여 에셋을 구성해야 합니다.

## 🎯 목표 (Objective)
Xcode의 `Assets.xcassets/AppIcon` 설정을 iOS 18 규격에 맞게 업데이트하여, 알림 아이콘에 강제로 씌워지는 흰색 테두리를 제거합니다.

## 🛠️ 작업 지시 사항 (Instructions)

### Step 1: 이미지 에셋 준비 가이드 (사용자/디자이너 수행 필요)
AI는 다음 규격에 맞는 이미지 에셋이 준비되었는지 확인하고, 없다면 사용자에게 요청하세요.
1. **Any (Light) / Dark 용 이미지:** 기존과 동일한 1024x1024 해상도의 꽉 찬(투명도 없는) 이미지
2. **Tinted 용 이미지 (핵심):** 1024x1024 해상도이며, **배경이 완전히 투명하고(Alpha 채널 포함) 로고(전경)만 단색(주로 흰색 또는 검은색)으로 존재하는 이미지**

### Step 2: Xcode AppIcon 설정 변경
1. `ios/Runner/Assets.xcassets/AppIcon.appiconset` 경로의 `Contents.json` 파일을 수정하거나, Xcode를 열어 설정을 변경합니다.
2. AppIcon의 **Appearance** 속성을 기존 `None` 또는 `Single`에서 **`Any, Dark, Tinted`**를 모두 지원하도록 변경합니다.
3. Xcode Inspector 기준: `AppIcon` 선택 -> 우측 패널 -> `iOS` 탭 -> `Appearance` 드롭다운에서 `Any, Dark, Tinted` 선택.

### Step 3: 에셋 매핑 및 Contents.json 업데이트
준비된 이미지를 각 슬롯에 맞게 매핑하도록 `Contents.json`을 업데이트하세요.
*   `Any` 슬롯: 기존 꽉 찬 이미지
*   `Dark` 슬롯: 다크 모드용 꽉 찬 이미지 (없으면 Any와 동일한 이미지 사용)
*   `Tinted` 슬롯: **배경이 투명한 전경(로고) 이미지**

**[Contents.json 수정 예시]**
```json
{
  "images" : [
    {
      "idiom" : "universal",
      "platform" : "ios",
      "size" : "1024x1024",
      "appearance" : [
        {
          "appearance" : "luminosity",
          "value" : "tinted"
        }
      ],
      "filename" : "Icon-Tinted.png" // 투명 배경의 로고 이미지
    },
    {
      "idiom" : "universal",
      "platform" : "ios",
      "size" : "1024x1024",
      "appearance" : [
        {
          "appearance" : "luminosity",
          "value" : "dark"
        }
      ],
      "filename" : "Icon-Dark.png" // 다크 모드용 꽉 찬 이미지
    },
    {
      "idiom" : "universal",
      "platform" : "ios",
      "size" : "1024x1024",
      "filename" : "Icon-Light.png" // 기존 꽉 찬 이미지
    }
  ],
  "info" : {
    "author" : "xcode",
    "version" : 1
  }
}
```

### Step 4: 검증 (Verification)
1. 변경 사항 적용 후 `flutter clean` 및 `flutter pub get`을 실행합니다.
2. iOS 기기(iOS 18 이상) 또는 시뮬레이터에서 앱을 빌드합니다.
3. 기기를 다크 모드로 설정한 후 알림을 발생시켜, 아이콘 주변의 흰색 테두리가 사라지고 투명 배경의 로고가 시스템 색상(Tint)에 맞게 자연스럽게 렌더링되는지 확인합니다.
