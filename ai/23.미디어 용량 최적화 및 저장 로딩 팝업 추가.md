## 작업 2. media_attachment_field.dart — 미디어 압축 적용

### 2-1. import 추가
파일 상단에 아래 두 import를 추가한다.

```
import 'package:flutter_image_compress/flutter_image_compress.dart';
import 'package:video_compress/video_compress.dart';
```

### 2-2. 카메라 촬영 시 해상도 제한
`_pickFromCamera()` 메서드 내부에서 `_picker.pickImage()`와 `_picker.pickVideo()` 호출 부분을 수정한다.

- 사진 촬영 시: `maxWidth: 1920`, `maxHeight: 1080`, `imageQuality: 85` 옵션을 추가한다.
- 동영상 촬영 시: `maxDuration: const Duration(minutes: 5)` 옵션을 추가한다.

### 2-3. 갤러리 선택 시 해상도 제한
`_pickFromGallery()` 메서드 내부에서 `_picker.pickMultipleMedia()` 호출 부분을 수정한다.

- `maxWidth: 1920`, `maxHeight: 1080`, `imageQuality: 85` 옵션을 추가한다.
- 단, `pickMultipleMedia()`가 해당 옵션을 지원하지 않을 경우, `pickMultiImage()`로 대체하여 동일한 옵션을 적용한다.

### 2-4. saveToLocalFiles() — 저장 시 압축 처리
`RecordMediaController`의 `saveToLocalFiles()` 메서드 내부, 파일을 복사하는 로직을 아래 기준으로 수정한다.

**사진인 경우 (`item.type == RecordMediaType.photo`):**
- `source.copy(destPath)` 대신 `FlutterImageCompress.compressAndGetFile()`을 사용한다.
- 옵션: `quality: 85`, `minWidth: 1920`, `minHeight: 1080`, `keepExif: false`
- 압축 결과가 `null`이면 해당 항목을 건너뛴다(`continue`).

**동영상인 경우 (`item.type == RecordMediaType.video`):**
- `source.copy(destPath)` 대신 `VideoCompress.compressVideo()`를 사용한다.
- 옵션: `quality: VideoQuality.Res1280x720Quality`, `frameRate: 30`, `deleteOrigin: false`
- 압축 결과의 `file`이 `null`이면 해당 항목을 건너뛴다(`continue`).
- 압축된 파일을 `destPath`로 복사한다.

> **주의:** `kIsWeb`이 `true`인 경우 기존과 동일하게 빈 리스트를 반환하고 압축 로직을 실행하지 않는다.

## 작업 3. record_edit_sheet.dart — 저장/수정 시 로딩 팝업 추가

### 3-1. 로딩 팝업 헬퍼 메서드 추가
`_RecordEditSheetState` 클래스 내부에 아래 두 메서드를 추가한다.

**로딩 팝업 표시 메서드 `_showSavingDialog()`:**
- `showDialog()`를 사용하여 팝업을 띄운다.
- `barrierDismissible: false`로 설정하여 사용자가 임의로 닫지 못하게 한다.
- 팝업 내용: `CircularProgressIndicator`와 함께 로컬라이징된 "저장 중..." 텍스트를 표시한다.
  - l10n 키가 없으면 `context.l10n.saving` 또는 임시로 하드코딩 문자열 `'저장 중...'`을 사용한다.
- Row 또는 Column으로 인디케이터와 텍스트를 나란히 배치한다.

**로딩 팝업 닫기 메서드 `_hideSavingDialog()`:**
- `Navigator.of(context, rootNavigator: true).pop()`을 호출하여 팝업을 닫는다.

### 3-2. _save() 메서드에 로딩 팝업 적용
기존 `_save()` 메서드의 흐름을 아래와 같이 수정한다.

1. 유효성 검사(각 category별 validation)가 모두 통과된 직후, `_mediaController.saveToLocalFiles()` 호출 **이전**에 `_showSavingDialog()`를 호출한다.
2. `saveToLocalFiles()` 및 `ref.read(recordNotifierProvider.notifier).update(updated)` 완료 후, `showTopToast` 호출 **이전**에 `_hideSavingDialog()`를 호출한다.
3. 예외가 발생하더라도 팝업이 남아있지 않도록, 전체 저장 로직을 `try/finally` 블록으로 감싸고 `finally`에서 `_hideSavingDialog()`를 호출한다.

### 3-3. 로딩 팝업 동작 흐름 요약
```
[저장 버튼 탭]
    → 유효성 검사
    → _showSavingDialog() ← 팝업 표시
    → saveToLocalFiles() (이미지/동영상 압축 포함, 시간 소요)
    → record update
    → _hideSavingDialog() ← 팝업 닫기
    → showTopToast("저장되었습니다")
    → Navigator.pop(context, true)
```

## 주의사항

- 압축 로직은 **`kIsWeb == false`** 일 때만 실행한다. 웹 환경에서는 기존 로직을 유지한다.
- `video_compress`는 iOS/Android 네이티브 의존성이 있으므로, 빌드 오류 발생 시 iOS는 `Podfile`, Android는 `build.gradle` 설정을 확인한다.
- `_hideSavingDialog()`는 반드시 `mounted` 체크 후 호출한다.
