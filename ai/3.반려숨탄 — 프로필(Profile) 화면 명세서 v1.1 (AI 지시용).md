## 1. 개요

### 1-1. 화면 목적
- 현재 선택된 펫의 기본 정보 및 식별 정보를 **인라인으로 확인 및 수정**
- 멀티펫 **전환** (헤더 칩 탭 → 바텀시트)
- **새 펫 추가** (최상단 버튼 또는 펫 전환 바텀시트 내 버튼)
- **펫 삭제** (하단 삭제 버튼 → 확인 다이얼로그)
- 앱 설정/기타는 이 화면에 **포함하지 않음** → 별도 설정(Settings) 화면에서 관리

## 3. 데이터 모델

### 3-1. `pets` 테이블 스키마
```sql
CREATE TABLE pets (
  id            INTEGER PRIMARY KEY AUTOINCREMENT,
  name          TEXT NOT NULL,
  species       TEXT,              -- 'dog' | 'cat' | 'other'
  breed         TEXT,
  birth_date    TEXT,              -- ISO8601 (YYYY-MM-DD)
  gender        TEXT,              -- 'male' | 'female'
  weight        REAL,              -- kg 단위
  is_neutered   INTEGER DEFAULT 0, -- 0: 미완료, 1: 완료
  microchip_id  TEXT,              -- 마이크로칩 번호 (최대 15자리)
  reg_number    TEXT,              -- 동물등록번호
  photo_path    TEXT,              -- 로컬 이미지 경로
  created_at    TEXT NOT NULL      -- ISO8601
)
```

### 3-2. Dart 모델
```dart
class Pet {
  final int?    id;
  final String  name;
  final String? species;      // 'dog' | 'cat' | 'other'
  final String? breed;
  final String? birthDate;    // YYYY-MM-DD
  final String? gender;       // 'male' | 'female'
  final double? weight;       // kg
  final bool    isNeutered;
  final String? microchipId;
  final String? regNumber;
  final String? photoPath;
  final String  createdAt;
}
```

### 3-3. 나이 자동 계산 (읽기 전용)
```dart
String get ageLabel {
  if (birthDate == null) return '미입력';
  final birth = DateTime.parse(birthDate!);
  final now   = DateTime.now();
  final totalMonths =
      (now.year - birth.year) * 12 + (now.month - birth.month);
  if (totalMonths < 12) return '${totalMonths}개월';
  final y = totalMonths ~/ 12;
  final m = totalMonths % 12;
  return m > 0 ? '${y}살 ${m}개월' : '${y}살';
}
```

## 4. 상태 관리

### 4-1. Provider 구조
```dart
// 전체 펫 목록
final petListProvider =
    StateNotifierProvider<PetNotifier, AsyncValue<List<Pet>>>(
      (ref) => PetNotifier(ref.watch(petRepositoryProvider)),
    );

// 현재 선택된 펫 ID (앱 전역 공유)
final selectedPetIdProvider = StateProvider<int?>((ref) => null);

// 현재 선택된 펫 (파생)
final currentPetProvider = Provider<Pet?>((ref) {
  final id   = ref.watch(selectedPetIdProvider);
  final pets = ref.watch(petListProvider).valueOrNull ?? [];
  return pets.firstWhereOrNull((p) => p.id == id);
});
```

### 4-2. PetNotifier 메서드 요약
| 메서드 | 동작 |
|--------|------|
| `add(Pet)` | DB INSERT → selectedPetId 자동 설정 → **체중 있으면 records 자동 기록** |
| `update(Pet, {double? previousWeight})` | DB UPDATE → **체중 변경 시 records 자동 기록** |
| `delete(int id)` | DB DELETE + alarms/records CASCADE → 다음 펫으로 selectedPetId 전환 |
| `switchPet(int id)` | selectedPetIdProvider 업데이트 |

## 5. 체중 → 일지 기록 자동 연동

### 5-1. 연동 규칙
| 상황 | 동작 |
|------|------|
| 새 펫 추가 + 체중 **입력됨** | `records` 테이블에 `type: 'weight'` INSERT |
| 새 펫 추가 + 체중 **미입력** | records 생성 안 함 |
| 프로필 수정 + 체중 **변경됨** | 이전 값과 다를 때만 records INSERT |
| 프로필 수정 + 체중 **동일** | records INSERT 금지 (중복 방지) |

### 5-2. records 테이블 체중 기록 스펙
```sql
INSERT INTO records (
  pet_id,
  type,        -- 'weight'
  value,       -- 체중 수치 REAL (kg)
  unit,        -- 'kg'
  recorded_at, -- 저장 시각 ISO8601
  source,      -- 'profile'  ← 프로필에서 자동 생성된 기록 식별용
  memo         -- NULL
)
```

### 5-3. PetNotifier 구현
```dart
/// 새 펫 추가
Future<void> add(Pet pet) async {
  final newId = await _repo.insert(pet);
  ref.read(selectedPetIdProvider.notifier).state = newId;

  if (pet.weight != null) {
    await _recordRepo.insert(Record(
      petId:      newId,
      type:       'weight',
      value:      pet.weight,
      unit:       'kg',
      recordedAt: DateTime.now().toIso8601String(),
      source:     'profile',
    ));
  }
  state = await _repo.findAll();
}

/// 프로필 수정
Future<void> update(Pet pet, {double? previousWeight}) async {
  await _repo.update(pet);

  final weightChanged = pet.weight != null && pet.weight != previousWeight;
  if (weightChanged) {
    await _recordRepo.insert(Record(
      petId:      pet.id!,
      type:       'weight',
      value:      pet.weight,
      unit:       'kg',
      recordedAt: DateTime.now().toIso8601String(),
      source:     'profile',
    ));
  }
  state = await _repo.findAll();
}
```

### 5-4. 체중 포커스 아웃 처리 (ProfileScreen)
```dart
void _onWeightUnfocus() {
  final newWeight      = double.tryParse(_weightController.text.trim());
  final previousWeight = ref.read(currentPetProvider)?.weight;
  if (newWeight == null) return;
  final updated = currentPet.copyWith(weight: newWeight);
  petNotifier.update(updated, previousWeight: previousWeight);
}
```

### 5-5. 일지 화면에서의 표시
- `source == 'profile'` 기록 → 부제목에 **"프로필에서 기록됨"** 표시
- 사용자가 일지에서 직접 수정/삭제 가능 (일반 기록과 동일하게 취급)

## 7. 자동 저장 방식 (C 방식 확정)

| 필드 | 저장 시점 | records 연동 |
|------|----------|-------------|
| 이름 / 품종 / 마이크로칩 / 등록번호 | 포커스 아웃 (`onEditingComplete`) | 없음 |
| **체중** | 포커스 아웃 (`onEditingComplete`) | ✅ 값 변경 시 records 자동 INSERT |
| 생년월일 | DatePicker 선택 완료 즉시 | 없음 |
| 성별 | 칩 탭 즉시 | 없음 |
| 중성화 | 토글 탭 즉시 | 없음 |

### 자동 저장 토스트
```dart
ScaffoldMessenger.of(context).showSnackBar(

    content: Text('✅  저장됐어요'),
    duration: Duration(seconds: 1),
    behavior: SnackBarBehavior.floating,
    shape: StadiumBorder(),
    backgroundColor: AppColors.gray800,
  ),
);
```

## 14. 구현 시 주의사항

1. **저장 버튼 위치** — `ProfileScreen`에 저장 버튼 추가 금지. `AddPetScreen` 하단에만 존재.
2. **자동 저장 C 방식** — TextInput: 포커스 아웃 시 저장 / DatePicker·GenderChip·Toggle: 변경 즉시 저장.
3. **저장 토스트** — 매 저장마다 `SnackBar(duration: 1s, floating, stadium)` 표시.
4. **체중 records 자동 기록** — 새 펫 추가 시 체중 입력됐으면 INSERT. 수정 시 이전 값과 다를 때만 INSERT. 동일하면 INSERT 금지.
5. **체중 records source** — `source: 'profile'` 로 저장. 일지에서 "프로필에서 기록됨" 부제목 표시.
6. **일지 체중 기록** — `source == 'profile'` 기록도 사용자가 수정/삭제 가능 (일반 기록과 동일 취급).
7. **나이 필드** — 읽기 전용. 생년월일 기반 자동 계산. ✏️ 아이콘 없음.
8. **새 펫 추가 헤더** — 뒤로가기 버튼 필수. 햄버거 버튼 없음.
9. **펫 전환** — 헤더 PetChip 탭 → PetSwitchSheet. 별도 탭바 항목 없음.
10. **새 펫 추가 진입 경로 2가지** — ① ProfileScreen 최상단 버튼, ② PetSwitchSheet 내 버튼.
11. **펫 삭제 시 캐스케이드** — alarms, records 테이블도 함께 삭제.
12. **펫 0마리일 때** — `/profile/add` 강제 이동. 뒤로가기 비활성화.
13. **마이크로칩 입력** — 숫자만 허용, 최대 15자리. `FilteringTextInputFormatter.digitsOnly`.
14. **체중 입력** — 소수점 허용. `TextInputType.numberWithOptions(decimal: true)`.
15. **생년월일** — `showDatePicker`. 미래 날짜 선택 불가 (`lastDate: DateTime.now()`).
16. **selectedPetId 전역 관리** — 알림, 일지, 홈 화면 모두 이 값 기준으로 필터링.
17. **앱 설정/기타** — 프로필 화면에 포함하지 않음. 별도 설정 화면에서 관리.
18. **펫 사진 dashed 버튼** — Flutter 기본 `OutlinedButton`은 dashed 미지원. `CustomPainter`로 구현.

## 15. 전역 (기본 레이아웃) 적용사항

모든 화면의 상단 메뉴의 헤더 PetChip의 기능은 항목 **11. 펫 전환 바텀시트 (PetSwitchSheet)** 기능으로 기본 적용.
