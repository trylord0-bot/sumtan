# 🥣💧 식사 · 음수 카테고리 분리 작업 지시서

## 개요
기존 `meal` 카테고리에 통합되어 있던 **식사**와 **음수**를 완전히 분리한다.
- `meal` → 고형 음식(사료·간식 등) 전용
- `water` → 음수 전용 (신규 카테고리)

> **중요**: 음수량 단계(정성)와 mL 입력값은 완전히 독립된 별개의 데이터로 취급한다.
> 단계값을 mL로 환산하거나, mL를 단계로 역산하는 로직은 절대 구현하지 않는다.

## 8. 주의사항

- 기존 `meal` 카테고리 데이터 마이그레이션 불필요 (음수 필드 제거만)
- `water` 카테고리는 신규이므로 기존 데이터에 영향 없음
- Firestore 사용 시 `category: 'water'` 필드로 구분
- **음수 색상은 반드시 Cyan(`#22D3EE`)을 사용한다 — 기존 9개 카테고리 색상과 중복 금지**
- **음수량 단계(waterAmount)와 mL(milliliter)는 항상 독립적으로 처리한다**
  - 단계 → mL 환산 금지
  - mL → 단계 역산 금지
  - 차트·통계는 단계 점수만 사용
  - mL는 기록 상세 표시 전용
