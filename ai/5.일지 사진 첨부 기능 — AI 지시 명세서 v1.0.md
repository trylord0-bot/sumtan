# 📷 반려숨탄 — 일지 미디어 첨부 기능 AI 지시 명세서 v2.0

## 📌 역할 지시

당신은 **반려숨탄 Flutter 앱의 기존 코드베이스를 완전히 파악한 모바일 앱 개발 전문가**입니다.
이 명세서는 **일지의 모든 카테고리(배변, 컨디션, 투약, 체중 등) 항목 추가/수정 팝업에 사진 및 동영상 첨부 기능을 추가**하기 위한 지시서입니다.
기존 코드 구조, 상태 관리(Riverpod), 라우팅(GoRouter), DB(sqflite) 패턴을 유지하면서 구현하세요.

> ⚠️ 이 명세서는 **UI/UX 동작 및 기능 범위**를 지시합니다.
> DB 스키마 설계, 파일 저장 구조 등 기술적 구현 방식은 기존 코드베이스의 패턴을 따르세요.

---

## 🎯 구현 목표

일지의 **모든 카테고리 항목 추가/수정 팝업** 하단에 사진과 동영상을 함께 첨부할 수 있는 기능을 추가한다.
카테고리 종류(배변, 컨디션, 투약, 체중 등)에 관계없이 **동일한 미디어 첨부 UI와 동작**이 적용된다.

## 🖼️ UI/UX 상세 명세

### 1. 팝업 내 미디어 첨부 영역

#### 위치
- 모든 카테고리의 추가/수정 팝업에서 **기존 입력 필드 전체의 최하단**에 배치
- 메모 입력 필드 아래, 저장 버튼 위에 위치

#### 레이아웃
- 섹션 레이블:
  - 미디어 없을 때: `사진 · 동영상`
  - 미디어 1개 이상: `사진 · 동영상 · N개`
- 썸네일과 `+` 버튼을 **Wrap 레이아웃**으로 가로 나열
- 한 줄 최대 **4개** (썸네일 3개 + `+` 버튼 1개 기준), 초과 시 다음 줄로 자동 줄바꿈
- 미디어 개수 제한: **무제한**

#### 상태별 표시

| 미디어 수 | 표시 |
|-----------|------|
| 0개 | `[+ 미디어 추가]` |
| 1개 | `[썸네일][+]` |
| 2개 | `[썸네일][썸네일][+]` |
| 3개 이상 | `[썸네일]...[썸네일][+]` (줄바꿈 포함) |

#### 공통 썸네일 스펙
- 크기: **72×72dp** 정사각형
- 모서리: 둥근 모서리 (10dp)
- 이미지: `object-fit: cover` (비율 유지 크롭)
- 좌하단에 **미디어 타입 배지** 표시
  - 사진: 📷 아이콘 (반투명 검정 배경)
  - 동영상: 🎬 아이콘 (반투명 검정 배경)
- 동영상 썸네일 추가 요소:
  - 첫 번째 프레임을 썸네일로 표시
  - 중앙에 **▶ 재생 아이콘** 오버레이 (반투명 흰 원형)

#### `+` 추가 버튼 스펙
- 크기: 썸네일과 동일한 **72×72dp** 정사각형
- 스타일: 점선 테두리(`gray300`) + 중앙 `+` 아이콘(`gray400`)
- 미디어 0개일 때: `+` 아이콘 아래 `미디어 추가` 레이블 함께 표시
- 탭 동작: **미디어 선택 바텀시트** 열기

---

### 2. 미디어 선택 바텀시트

#### 진입 조건
- 팝업 내 `+` 버튼 탭 시 표시

#### 구성 (위에서 아래 순서)
1. 핸들 바 (상단 중앙)
2. 타이틀: `미디어 추가 방법 선택` (회색 소문자)
3. **카메라로 촬영** 항목
   - 아이콘: 📷 (초록 원형 배경 `primary100`)
   - 텍스트: `카메라로 촬영`
   - 부제목: `사진 또는 동영상 촬영`
4. 구분선
5. **갤러리에서 선택** 항목
   - 아이콘: 🖼️ (파란 원형 배경 `catMed`)
   - 텍스트: `갤러리에서 선택`
   - 부제목: `사진 · 동영상 다중 선택 가능`
6. **취소** 버튼 (회색 배경, 별도 버튼 스타일)

#### 동작
- **카메라 선택**: 시스템 카메라 앱 실행 → 사진 또는 동영상 촬영 선택은 시스템에 위임 → 촬영 완료 후 팝업으로 복귀, 썸네일 추가
- **갤러리 선택**: 갤러리 앱 실행 → **사진/동영상 다중 선택** 후 팝업으로 복귀, 썸네일 일괄 추가
- **취소**: 바텀시트 닫힘, 기존 상태 유지

#### 권한 처리
- 권한 미허용 시: 권한 요청 다이얼로그 표시
- 권한 영구 거부 시: `"설정에서 권한을 허용해 주세요"` 안내 + 설정 화면 이동 버튼

---

### 3. 사진 프리뷰 팝업

#### 진입 조건
- 팝업 내 **사진 썸네일** 탭 시 표시

#### 구성
- 배경: 검정 (`#000000`)
- 상단 우측: **닫기 버튼** (반투명 원형 배경 + ✕ 아이콘)
- 중앙: 사진 원본 표시 (가로 전체 너비, 비율 유지)
- 하단 중앙: **`🗑️ 이 사진 삭제`** 버튼 (빨간 pill 형태)
- **핀치줌(확대/축소)** 지원

#### 동작
- 닫기 버튼 탭 또는 뒤로가기: 프리뷰 팝업 닫힘, 기존 팝업으로 복귀
- 삭제 버튼 탭: **삭제 확인 다이얼로그** 표시

---

### 4. 동영상 프리뷰 팝업 (인라인 플레이어)

#### 진입 조건
- 팝업 내 **동영상 썸네일** 탭 시 표시

#### 사용 패키지
- `video_player` (Flutter 공식 패키지)

#### 구성
- 배경: 검정 (`#000000`)
- 상단 우측: **닫기 버튼** (반투명 원형 배경 + ✕ 아이콘)
- 중앙: 동영상 인라인 재생 영역 (16:9 비율, 가로 전체 너비)
- 하단 중앙: **`🗑️ 이 동영상 삭제`** 버튼 (빨간 pill 형태)

#### 자동 재생
- 프리뷰 팝업 진입 즉시 **자동으로 재생 시작**
- 초기 상태: **음소거(mute)** 로 시작

#### 컨트롤바 동작 (C안 — 자동 페이드)

| 상태 | 컨트롤바 표시 여부 |
|------|-----------------|
| 재생 시작 직후 | 컨트롤바 표시 |
| 재생 중 3초 경과 | 컨트롤바 + 상단 닫기 버튼 + 하단 삭제 버튼 **페이드 아웃** |
| 화면 탭 | 컨트롤바 + 상단/하단 버튼 **페이드 인**, 3초 후 다시 페이드 아웃 |
| 일시정지 | 컨트롤바 항상 표시 유지 (페이드 아웃 안 됨) + 중앙 ▶ 아이콘 표시 |

#### 컨트롤바 구성 (하단 오버레이, 그라데이션 배경)
1. **진행 바**
   - 현재 재생 위치 표시 (초록색 `primary`)
   - 진행 바 끝에 흰색 원형 핸들
   - 좌측: 현재 재생 시간 (`0:00` 형식)
   - 우측: 전체 재생 시간 (`0:00` 형식)
   - 드래그로 탐색(seek) 가능
2. **하단 버튼 행**
   - 좌측: ▶ / ⏸ 재생·일시정지 토글 버튼
   - 우측: 🔇 음소거 / 🔊 소리 켜기 토글 버튼 (pill 형태)

#### 닫기 동작
- 닫기 버튼 탭 또는 뒤로가기: 재생 중지 후 팝업 닫힘, 기존 팝업으로 복귀

#### 삭제 버튼 탭 동작
- 재생 일시정지 후 **삭제 확인 다이얼로그** 표시

---

### 5. 삭제 확인 다이얼로그

#### 구성
- 타이틀:
  - 사진 삭제 시: `사진을 삭제할까요?`
  - 동영상 삭제 시: `동영상을 삭제할까요?`
- 부가 설명: **없음**
- 버튼 2개 (가로 나열):
  - `취소` (회색 배경)
  - `삭제` (빨간 배경)

#### 동작
- **취소** 탭: 다이얼로그 닫힘, 프리뷰 팝업 유지 (동영상은 재생 재개)
- **삭제** 탭: 해당 썸네일 제거, 프리뷰 팝업 닫힘, 기존 팝업으로 복귀

> ⚠️ 프리뷰에서의 삭제는 **팝업 저장 전까지 임시 상태**입니다.
> 팝업을 저장해야 최종 반영됩니다.

---

### 6. 저장 동작

#### 팝업 저장 버튼 탭 시
1. 선택된 미디어 파일들을 **앱 전용 로컬 디렉토리**에 복사 저장
2. 저장된 파일 경로 및 미디어 타입(사진/동영상)을 해당 일지 항목과 함께 DB에 저장
3. 파일명과 DB 레코드는 항상 **1:1 페어로 유지**

#### 팝업 취소(닫기) 시
- 아직 저장되지 않은 임시 선택 미디어는 로컬에 복사하지 않음
- 기존에 저장된 미디어는 변경 없이 유지

#### 항목 수정 시 팝업 재진입
- 기존에 저장된 미디어가 썸네일로 **복원되어 표시**되어야 함
- 기존 미디어 삭제 및 신규 미디어 추가 모두 가능

---

### 7. 항목 삭제 시

- 해당 항목과 연결된 **모든 미디어 파일을 로컬에서 자동 삭제**
- DB 레코드 삭제와 파일 삭제는 **반드시 함께 처리** (고아 파일 방지)

## 📤 Export 기능

### 방식
- 일지 데이터를 내보낼 때 **JSON 데이터 파일 + 미디어 파일을 하나의 zip 파일로 묶어 내보내기**

### zip 내부 구조
```
sumtan_export_YYYYMMDD.zip
├── data.json              ← 전체 일지 데이터 (미디어 파일명 및 타입 참조 포함)
└── media/
    ├── {uuid}.jpg         ← 사진
    ├── {uuid}.png         ← 사진
    ├── {uuid}.mp4         ← 동영상
    └── ...
```

### 데이터-파일 일관성 규칙
- JSON 내 각 항목은 첨부 미디어의 **파일명(또는 상대 경로)과 타입(photo/video)을 참조**
- zip 내 `media/` 폴더의 파일명과 JSON 참조값이 **반드시 일치**
- Import(복원) 시 동일한 구조로 복원 가능하도록 설계

## 🎨 디자인 시스템 적용

| 요소 | 적용 스펙 |
|------|----------|
| `+` 버튼 테두리 | `gray300` 점선 |
| `+` 버튼 아이콘 | `gray400` |
| 카메라 아이콘 배경 | `primary100` (`#D1FAE5`) |
| 갤러리 아이콘 배경 | `catMed` (`#DBEAFE`) |

| 프리뷰 배경 | `#000000` |
| 닫기 버튼 배경 | `rgba(255,255,255,0.15)` |
| 진행 바 색상 | `primary` (`#10B981`) |
| 컨트롤바 배경 | `linear-gradient(transparent → rgba(0,0,0,0.75))` |

- 바텀시트, 다이얼로그 등 공통 컴포넌트는 **기존 앱 내 동일 컴포넌트 스타일 유지**
