## 🔴 Critical — 즉시 수정 필요

### 1. `flutter_local_notifications` 패키지 누락

**문제**: `alarms` 테이블이 DB에 존재하고 `scheduled_at`, `repeat_rule`, `repeat_time` 등 알람 스케줄 필드가 있으나, 실제 OS 알림을 발송할 패키지가 `pubspec.yaml`에 없음.

**AI 수정 지시**:
```
pubspec.yaml의 dependencies에 아래 패키지를 추가하라:

  flutter_local_notifications: ^17.0.0
  timezone: ^0.9.0

그리고 Android의 AndroidManifest.xml에 아래 권한을 추가하라:
  <uses-permission android:name="android.permission.RECEIVE_BOOT_COMPLETED"/>
  <uses-permission android:name="android.permission.SCHEDULE_EXACT_ALARM"/>
  <uses-permission android:name="android.permission.POST_NOTIFICATIONS"/>

iOS의 Info.plist에 아래를 추가하라:
  <key>UIBackgroundModes</key>
  <array><string>fetch</string><string>remote-notification</string></array>
```

### 2. `DatabaseHelper` — 타입 미지정 (암묵적 dynamic)

**문제**: `Future get database`, `Future _init()`, `Future _onCreate(...)`, `Future _onUpgrade(...)`, `Future _createAlarmsTable(...)` 모두 반환 타입이 명시되지 않아 `dynamic`으로 처리됨. 타입 안전성 저하 및 런타임 오류 위험.

> ✅ **확인**: `import 'package:sqflite_common/sqflite.dart';`는 이미 코드 상단에 존재하므로 별도 추가 불필요.

**AI 수정 지시**:
```
database_helper.dart의 모든 Future 반환 타입을 아래와 같이 명시하라:

- Future get database          →  Future<Database> get database
- Future _init()               →  Future<Database> _init()
- Future _onCreate(Database db, int version)       →  Future<void> _onCreate(...)
- Future _onUpgrade(Database db, int old, int new) →  Future<void> _onUpgrade(...)
- Future _createAlarmsTable(Database db)           →  Future<void> _createAlarmsTable(...)

※ sqflite_common import는 이미 존재하므로 추가 불필요.
```

### 3. `DatabaseHelper` — Singleton `_db` null 체크 경쟁 조건 (Race Condition)

**문제**: `_db ??= await _init()` 패턴은 비동기 환경에서 동시에 여러 호출이 발생할 경우 `_init()`이 중복 실행될 수 있음.

**AI 수정 지시**:
```
database_helper.dart의 database getter를 아래와 같이 Completer 패턴으로 교체하라:

Completer<Database>? _dbCompleter;

Future<Database> get database async {
  if (_db != null) return _db!;
  if (_dbCompleter != null) return _dbCompleter!.future;
  _dbCompleter = Completer<Database>();
  _db = await _init();
  _dbCompleter!.complete(_db);
  return _db!;
}

상단에 아래 import를 추가하라:
  import 'dart:async';
```

### 4. `onUpgrade` — 컬럼 중복 추가 위험 (`oldVersion < 4` 및 `oldVersion < 3` 블록 모두 해당)

**문제**:
- `oldVersion < 4` 블록: `ALTER TABLE pets ADD COLUMN weight REAL` 등을 실행하는데, 이미 `_onCreate`의 `pets` 테이블 DDL에 `weight REAL`, `microchip_id TEXT`, `reg_number TEXT`가 포함되어 있음.
- `oldVersion < 3` 블록: `ALTER TABLE alarms ADD COLUMN alarm_days TEXT`를 실행하는데, 이미 `_createAlarmsTable`의 DDL에 `alarm_days TEXT`가 포함되어 있음.

신규 설치 후 업그레이드 경로를 타면 두 블록 모두 중복 컬럼 오류 발생 가능.

**AI 수정 지시**:
```
_onUpgrade 내부의 oldVersion < 3 블록과 oldVersion < 4 블록을 아래와 같이 수정하라.
이미 _onCreate / _createAlarmsTable에 포함된 컬럼이므로 try-catch로 예외 처리하라:

if (oldVersion < 3) {
  try {
    await db.execute('ALTER TABLE alarms ADD COLUMN alarm_days TEXT');
  } catch (_) {}
}

if (oldVersion < 4) {
  try {
    await db.execute('ALTER TABLE pets ADD COLUMN weight REAL');
  } catch (_) {}
  try {
    await db.execute('ALTER TABLE pets ADD COLUMN microchip_id TEXT');
  } catch (_) {}
  try {
    await db.execute('ALTER TABLE pets ADD COLUMN reg_number TEXT');
  } catch (_) {}
}

장기적으로는 마이그레이션 전용 클래스(DatabaseMigration)를 분리하는 것을 권장.
```

### 5. `DatabaseHelper` — `PRAGMA foreign_keys` 미설정으로 `ON DELETE CASCADE` 무효

**문제**: SQLite는 기본적으로 외래키 제약이 **비활성화** 상태임. `records`, `alarms` 테이블에 `ON DELETE CASCADE`가 선언되어 있으나, `onConfigure`에서 `PRAGMA foreign_keys = ON`을 설정하지 않으면 **실제로 CASCADE가 동작하지 않음**. 이는 반려동물 삭제 시 관련 기록/알람이 삭제되지 않는 데이터 정합성 버그를 유발함.

**AI 수정 지시**:
```
database_helper.dart의 _init() 내 OpenDatabaseOptions에 onConfigure 콜백을 추가하라:

options: OpenDatabaseOptions(
  version: 4,
  onConfigure: (db) async {
    await db.execute('PRAGMA foreign_keys = ON');
  },
  onCreate: _onCreate,
  onUpgrade: _onUpgrade,
),

이 설정이 없으면 ON DELETE CASCADE 선언이 있어도 실제로 동작하지 않음.
```

### 6. `main.dart` — DB 초기화 오류 처리 누락

**문제**: `await DatabaseHelper.instance.database`가 실패할 경우 앱이 블랙스크린으로 크래시됨. 오류 처리 및 사용자 안내 화면이 없음.

**AI 수정 지시**:
```
main.dart를 아래와 같이 수정하라:

void main() async {
  WidgetsFlutterBinding.ensureInitialized();
  await initializeDateFormatting('ko');

  try {
    await DatabaseHelper.instance.database;
  } catch (e, stack) {
    // 오류 로깅 (향후 Firebase Crashlytics 연동 권장)
    debugPrint('DB 초기화 실패: $e\n$stack');
    runApp(const DbErrorApp());
    return;
  }

  runApp(const ProviderScope(child: SumtanApp()));
}

DbErrorApp은 사용자에게 오류 메시지를 표시하는 최소한의 MaterialApp 위젯으로 구현하라.
```

## 🟠 Warning — 구조 개선 권장

### 7. `SumtanApp`이 `ConsumerWidget`을 사용하지 않음

**문제**: `app.dart`의 `SumtanApp`이 `StatelessWidget`을 상속하고 있으나, Riverpod `ProviderScope` 하위에서 테마나 로케일을 동적으로 제어하려면 `ConsumerWidget`으로 변경해야 함.

**AI 수정 지시**:
```
app/app.dart에서 SumtanApp 클래스를 아래와 같이 변경하라:

// Before
class SumtanApp extends StatelessWidget { ... }

// After
class SumtanApp extends ConsumerWidget {
  const SumtanApp({super.key});

  @override
  Widget build(BuildContext context, WidgetRef ref) {
    // 향후 테마 Provider, 로케일 Provider 구독 가능
    return MaterialApp.router(
      title: '반려숨탄',
      theme: AppTheme.light,
      routerConfig: appRouter,
      debugShowCheckedModeBanner: false,
    );
  }
}

상단에 아래 import를 추가하라:
  import 'package:flutter_riverpod/flutter_riverpod.dart';
```

### 8. `pubspec.yaml` — 패키지 버전 범위 불일치 위험

**문제**: `sqflite ^2.3.0`과 `sqflite_common ^2.5.0`의 minor 버전이 다름. `sqflite_common_ffi ^2.3.0`과도 불일치. 의존성 충돌 가능성 있음.

**AI 수정 지시**:
```
pubspec.yaml의 sqflite 관련 패키지 버전을 아래와 같이 통일하라:

  sqflite: ^2.3.3
  sqflite_common: ^2.5.4
  sqflite_common_ffi: ^2.3.3
  sqflite_common_ffi_web: ^1.0.2

변경 후 반드시 flutter pub upgrade --major-versions 를 실행하고
pubspec.lock을 커밋에 포함하라.
```

**AI 수정 지시**:

```
웹 지원을 유지한다면 현재 구성(Flutter 3.x 자동 포함)으로 충분하며
```

### 12. `records` 테이블 — `data_json` 컬럼 구조 문서화 필요

**문제**: `records` 테이블의 `data_json TEXT` 컬럼은 JSON 문자열을 저장하는 것으로 보이나, 어떤 구조의 JSON인지 코드 내 문서화가 없음. 유지보수 시 혼란 야기.

**AI 수정 지시**:
```

각 category별 data_json 스키마를 주석 또는 Dart 모델 클래스로 문서화하라.

예시:
/// records.data_json 구조 (category별)
/// - 'weight'   : {"value": 3.5, "unit": "kg"}
/// - 'vaccine'  : {"vaccine_name": "종합백신", "next_date": "2026-06-01"}
/// - 'hospital' : {"clinic": "행복동물병원", "diagnosis": "피부염"}
```
