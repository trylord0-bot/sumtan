# 🔔 알림 미작동 문제 수정 지시서
> **프로젝트**: sumtan_app (반려숨탄 — 반려동물 건강관리 Flutter 앱)  
> **기술스택**: Flutter, Riverpod, SQLite(sqflite), go_router, shared_preferences  
> **알림 방식**: FCM/Firebase 미사용 → 로컬 알림 자체 구현 (`features/alarm/`)  
> **문제**: 알림 설정을 완료했으나 실제 알림이 발생하지 않음

```
    └── app/
        └── src/main/AndroidManifest.xml  ← Android 권한 선언
```

## 🔍 1단계: 현재 알림 구현 방식 파악

### 1-1. pubspec.yaml 확인

`pubspec.yaml`을 열어 알림 관련 패키지가 있는지 확인하라.

**현재 확인된 패키지 목록 (알림 관련 패키지 없음):**
```yaml
dependencies:
  flutter_riverpod: ^2.5.0
  sqflite: ^2.3.0
  shared_preferences: ^2.2.0
  go_router: ^13.0.0
  # ❌ flutter_local_notifications 없음
  # ❌ awesome_notifications 없음
  # ❌ firebase_messaging 없음
```

> ⚠️ **알림 전송을 위한 패키지가 pubspec.yaml에 존재하지 않을 가능성이 높다.**  
> 이것이 알림이 오지 않는 근본 원인일 수 있다.

**→ 확인 후 패키지가 없다면 [2단계 A]로, 있다면 [2단계 B]로 이동하라.**

### A-3. 알림 서비스 파일 생성

`lib/core/services/notification_service.dart` 파일을 **새로 생성**하라:

```dart
import 'package:flutter_local_notifications/flutter_local_notifications.dart';
import 'package:timezone/timezone.dart' as tz;
import 'package:timezone/data/latest_all.dart' as tz;

class NotificationService {
  static final NotificationService _instance = NotificationService._internal();
  factory NotificationService() => _instance;
  NotificationService._internal();

  final FlutterLocalNotificationsPlugin _plugin =
      FlutterLocalNotificationsPlugin();

  /// 앱 시작 시 반드시 호출
  Future<void> init() async {
    tz.initializeTimeZones();
    tz.setLocalLocation(tz.getLocation('Asia/Seoul'));

    const androidSettings =
        AndroidInitializationSettings('@mipmap/ic_launcher');
    const iosSettings = DarwinInitializationSettings(
      requestAlertPermission: true,
      requestBadgePermission: true,
      requestSoundPermission: true,
    );
    const settings = InitializationSettings(
      android: androidSettings,
      iOS: iosSettings,
    );

    await _plugin.initialize(settings);

    // Android 알림 채널 생성
    const channel = AndroidNotificationChannel(
      'sumtan_alarm_channel',
      '반려숨탄 알림',
      description: '반려동물 건강관리 알림',
      importance: Importance.max,
    );
    await _plugin
        .resolvePlatformSpecificImplementation<
            AndroidFlutterLocalNotificationsPlugin>()
        ?.createNotificationChannel(channel);
  }

  /// 권한 요청 (Android 13+ 필수)
  Future<bool> requestPermission() async {
    final android = _plugin
        .resolvePlatformSpecificImplementation<
            AndroidFlutterLocalNotificationsPlugin>();
    final granted = await android?.requestNotificationsPermission();
    return granted ?? false;
  }

  /// 매일 특정 시간에 반복 알림 예약
  Future<void> scheduleDailyAlarm({
    required int id,
    required String title,
    required String body,
    required int hour,
    required int minute,
  }) async {
    await _plugin.zonedSchedule(
      id,
      title,
      body,
      _nextInstanceOfTime(hour, minute),
      const NotificationDetails(
        android: AndroidNotificationDetails(
          'sumtan_alarm_channel',
          '반려숨탄 알림',
          importance: Importance.max,
          priority: Priority.high,
        ),
        iOS: DarwinNotificationDetails(),
      ),
      androidScheduleMode: AndroidScheduleMode.exactAllowWhileIdle,
      uiLocalNotificationDateInterpretation:
          UILocalNotificationDateInterpretation.absoluteTime,
      matchDateTimeComponents: DateTimeComponents.time, // 매일 반복
    );
  }

  /// 알림 취소
  Future<void> cancelAlarm(int id) async {
    await _plugin.cancel(id);
  }

  /// 모든 알림 취소
  Future<void> cancelAll() async {
    await _plugin.cancelAll();
  }

  tz.TZDateTime _nextInstanceOfTime(int hour, int minute) {
    final now = tz.TZDateTime.now(tz.local);
    var scheduled = tz.TZDateTime(tz.local, now.year, now.month, now.day, hour, minute);
    if (scheduled.isBefore(now)) {
      scheduled = scheduled.add(const Duration(days: 1));
    }
    return scheduled;
  }
}
```

## 🛠️ 2단계 B: 알림 패키지는 있으나 동작하지 않는 경우

`features/alarm/` 내 기존 코드를 열어 아래 항목을 순서대로 점검하라.

### B-1. 알림 플러그인 초기화 여부 확인

```dart
// ❌ 잘못된 예 — init()을 호출하지 않음
final plugin = FlutterLocalNotificationsPlugin();
plugin.show(...); // 초기화 없이 바로 사용

// ✅ 올바른 예
await plugin.initialize(settings); // 먼저 초기화
plugin.show(...);
```

### B-2. zonedSchedule vs show 사용 확인

- `show()` : 즉시 알림 (예약 불가)
- `zonedSchedule()` : 특정 시간 예약 알림 ← **반드시 이것을 사용해야 함**

```dart
// ❌ show()로는 예약 알림 불가
await plugin.show(0, '제목', '내용', details);

// ✅ 예약 알림은 zonedSchedule() 사용
await plugin.zonedSchedule(
  0, '제목', '내용',
  scheduledDate,
  details,
  androidScheduleMode: AndroidScheduleMode.exactAllowWhileIdle,
  uiLocalNotificationDateInterpretation:
      UILocalNotificationDateInterpretation.absoluteTime,
);
```

### B-3. 알림 채널 ID 불일치 확인

채널 생성 시 사용한 ID와 알림 발송 시 사용하는 ID가 **정확히 일치**하는지 확인하라.

```dart
// 채널 생성
const channel = AndroidNotificationChannel('sumtan_alarm_channel', ...);

// 알림 발송 — 반드시 동일한 ID 사용
const AndroidNotificationDetails('sumtan_alarm_channel', ...);
//                                ↑ 반드시 일치
```

### B-4. 알림 설정 저장 후 실제 등록 여부 확인

알림 설정을 DB(sqflite)나 shared_preferences에 저장만 하고,  
실제 `zonedSchedule()`을 **호출하지 않는 경우**가 많다.

```dart
// ❌ 저장만 하고 등록 안 함
await db.insert('alarms', alarmData);

// ✅ 저장 + 실제 알림 등록 모두 해야 함
await db.insert('alarms', alarmData);
await NotificationService().scheduleDailyAlarm(
  id: alarmId,
  title: '반려동물 건강 알림',
  body: alarmMessage,
  hour: hour,
  minute: minute,
);
```

## 🔄 3단계: 앱 재시작 시 알림 재등록

Android는 기기 재부팅 또는 앱 재설치 시 예약된 알림이 **모두 초기화**된다.  
`main.dart` 또는 앱 초기화 시점에 DB에서 알림 목록을 불러와 재등록하는 로직을 추가하라.

```dart
// main.dart 또는 앱 초기화 위치에 추가
Future<void> _restoreAlarms() async {
  final alarms = await AlarmRepository().getAllActiveAlarms(); // DB에서 활성 알림 조회
  for (final alarm in alarms) {
    await NotificationService().scheduleDailyAlarm(
      id: alarm.id,
      title: alarm.title,
      body: alarm.body,
      hour: alarm.hour,
      minute: alarm.minute,
    );
  }
}
```

## ✅ 4단계: 수정 후 검증

| 테스트 항목 | 확인 방법 |
|---|---|
| 알림 권한 허용 여부 | 기기 설정 → 앱 → sumtan_app → 알림 → 허용 확인 |
| 즉시 알림 동작 | `show()`로 즉시 알림 테스트 발송 |
| 예약 알림 동작 | 현재 시각 + 1분 후로 `zonedSchedule()` 테스트 |
| 백그라운드 동작 | 앱을 홈으로 내린 후 알림 수신 확인 |
| 재부팅 후 동작 | 기기 재시작 후 알림 재등록 로직 동작 확인 |
| Android 13+ 권한 | `requestNotificationsPermission()` 결과 로그 확인 |

## ⚠️ 주의사항

1. **핫 리스타트 금지** — 알림 초기화 변경 후에는 `flutter run`으로 완전 재빌드
2. **Android 12+에서 정확한 알람 사용 시** `SCHEDULE_EXACT_ALARM` 권한 필요  
   → 기기 설정 → 앱 → 특별한 앱 접근 권한 → 알람 및 리마인더에서 허용 필요
3. **timezone 패키지 초기화** — `tz.initializeTimeZones()` 없이 `zonedSchedule()` 사용 시 크래시 발생
