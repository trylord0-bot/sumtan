# 알림 기능 개선 지시서

> 대상 프로젝트: `sumtan_app` (Flutter + Riverpod + SQLite)
> 작성일: 2026-05-07

---

## 배경 및 목적

현재 알림 리스트에서 **오늘 예정된 알림(todayPending)** 카드는 완료 처리와 나중에 버튼만 존재하며,
내용을 수정하거나 알림 시간을 미루는 기능이 없다.
사용자가 알림을 더 유연하게 관리할 수 있도록 두 가지 기능을 추가한다.

---

## 기능 1. 예정된 알림 카드 터치 시 수정 팝업

### 요구사항

- 알림 리스트에서 **오늘 예정된 알림(todayPending) 카드**를 터치하면
  기존 알림 추가 시 사용하는 **폼 바텀시트(AlarmFormSheet)** 가 열려야 한다.
- 바텀시트는 **수정 모드**로 열리며, 해당 알림의 기존 정보(제목, 날짜, 시간, 메모 등)가
  모두 채워진 상태로 표시되어야 한다.
- 사용자가 내용을 변경하고 저장하면 알림 정보가 업데이트된다.
- 저장 성공 시 기존과 동일하게 상단 토스트 메시지로 피드백을 제공한다.

### 터치 영역 처리 주의사항

- 카드 하단의 **"✅ 완료했어요"** 버튼과 **"🕐 나중에"** 버튼 영역은
  카드 전체 터치 이벤트와 **겹치지 않도록** 처리해야 한다.
- 즉, 두 버튼을 터치했을 때는 수정 팝업이 열리지 않고 각 버튼 고유의 동작만 실행되어야 한다.
- 카드의 버튼 영역 **외** 나머지 영역(제목, 날짜, 아이콘 등)을 터치했을 때만 수정 팝업이 열린다.

### 참고

- 이미 `past` 상태의 알림은 `AlarmListItem` 위젯에서 터치 시 `isReschedule: true` 모드로
  `AlarmFormSheet`를 여는 패턴이 구현되어 있다. 동일한 방식을 참고하여 구현한다.
- `todayPending` 알림은 재예약이 아닌 **일반 수정 모드**(`isReschedule: false`)로 열면 된다.

## 기능 2. "나중에" 버튼 — 스누즈 시간 선택 팝업

### 현재 상태

현재 "나중에" 버튼은 토스트 메시지(`"나중에 다시 확인해드릴게요 🕐"`)만 표시하고
실제로 아무런 동작을 하지 않는다.

### 요구사항

- "나중에" 버튼을 터치하면 **스누즈 시간을 선택하는 바텀시트 팝업**이 열려야 한다.
- 팝업에는 아래 시간 옵션들이 선택지로 표시된다.

  | 선택지 | 미루는 시간 |
  |--------|------------|
  | 5분 뒤 | +5분 |
  | 10분 뒤 | +10분 |
  | 30분 뒤 | +30분 |
  | 1시간 뒤 | +60분 |
  | 3시간 뒤 | +180분 |
  | 내일 같은 시간 | +24시간 |

- 사용자가 시간 옵션을 선택하면 해당 알림의 `scheduledAt`을
  **현재 시각 기준으로 선택한 시간만큼 더한 값**으로 업데이트한다.
- 업데이트 후 알림 상태(`status`)가 자동으로 재계산되어 리스트에 반영된다.
  - 예: 5분 뒤, 10분 뒤, 30분 뒤 선택 시 → 여전히 오늘이므로 `todayPending` 유지
  - 예: 내일 같은 시간 선택 시 → `upcoming` 상태로 변경되어 일반 예정 알림 카드로 이동
- 저장 성공 시 상단 토스트로 피드백을 제공한다.
  - 예: `"🕐 30분 뒤에 다시 알려드릴게요"`

### Provider 연동

- `AlarmNotifier`에 `snooze(Alarm alarm, int minutes)` 메서드를 추가한다.
- 해당 메서드는 다음을 수행한다.
  1. 현재 `scheduledAt`을 기준으로 `minutes`만큼 더한 새로운 `scheduledAt`을 계산한다.
     - `scheduledAt`이 null이거나 이미 과거인 경우, **현재 시각**을 기준으로 계산한다.
  2. `alarm.copyWith(scheduledAt: 새로운시간)`으로 알림 객체를 갱신한다.
  3. DB에 업데이트하고 알림 스케줄을 재등록한다.
  4. 리스트를 리로드한다.

### UI 구성 가이드

- 바텀시트 상단에 알림 이름과 타입 이모지를 표시하여 어떤 알림을 미루는지 사용자가 확인할 수 있게 한다.
- 시간 옵션은 리스트 또는 그리드 형태로 나열하며, 기존 앱 디자인 시스템(AppColors, AppSpacing 등)을 따른다.
- 옵션 선택 즉시 바텀시트가 닫히고 토스트가 표시된다. (별도 확인 버튼 불필요)

## 공통 유의사항

- 두 기능 모두 기존 `AlarmFormSheet`, `AlarmNotifier`, `AlarmRepository` 구조를 최대한 재사용한다.
- 새로운 파일이 필요한 경우 기존 폴더 구조(`presentation/widgets/`, `provider/`)를 따른다.
- 상태 관리는 기존 Riverpod 패턴(`StateNotifier`, `ref.read(alarmListProvider.notifier)`)을 그대로 따른다.
- 알림 스케줄링 서비스(`NotificationService`)는 현재 stub 상태이므로 인터페이스 호출은 유지하되 실제 푸시 동작은 추후 구현 예정이다.
