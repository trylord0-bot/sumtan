## 2. 마일스톤 전략 개요

기능이 방대한 만큼, **3단계 점진적 배포 전략**을 채택한다.
각 마일스톤은 독립적으로 완결성을 가지며, 이전 버전 사용자가 자연스럽게 다음 버전으로 전환될 수 있도록 설계한다.

| 마일스톤 | 타겟 | 핵심 키워드 | 예상 개발 기간 |
|---------|------|------------|--------------|
| **1차 (MVP)** | 초보 보호자 | 기록 습관 형성 | 3~4개월 |
| **2차** | 일반 보호자 | 관리 고도화 | 3~4개월 |
| **3차** | 전문/파워 유저 | 연결 & 확장 | 4~6개월 |

## 3. 핵심 설계 원칙 (1차)

1. **로컬 우선**: 회원가입/로그인 없이 기기 로컬 DB에 데이터 저장
2. **이벤트 기반 기록**: 모든 기록은 이벤트 단위로 날짜에 연결
3. **단순한 UX 우선**: 초보자도 직관적으로 사용할 수 있도록 UI는 최대한 단순하게 설계
4. **빠른 기록**: FAB 버튼으로 홈에서 오늘 기록 즉시 접근
5. **날짜 기반 조회**: 캘린더에서 날짜 선택 → 과거 기록 조회 및 수정
6. **추세 파악**: 홈 화면 카드 형태로 체중/배변/컨디션 추세 그래프 제공
7. **알림 자율 설정**: 시스템 자동 알림이 아닌, 사용자가 직접 시간/날짜 지정
8. **응급 대응**: 응급 화면 원터치 접근 및 주변 병원 빠른 연결
9. **확장성 있는 Export**: key-value 기반 JSON 포맷으로 버전 간 호환성 확보

## 5. 1차 포함 기능 목록 (상세)

### 5-1. 반려동물 프로필

- 동물 종류, 이름, 품종, 생년월일, 성별, 중성화 여부 등록
- 프로필 사진 등록
- 다두 등록 및 프로필 전환 (1마리 무료 / 추가 마리당 유료)
- 품종별 표준 체중 / 평균 수명 기본 정보 제공

> **초보자 배려**: 등록 시 각 항목에 "왜 이 정보가 필요한가요?" 툴팁 제공

---

### 5-2. 날짜별 건강 일지 ⭐ 핵심

- 캘린더 뷰 제공
- 날짜별 컬러 인디케이터 표시
  - 🟢 좋음 / 🟡 보통 / 🔴 나쁨
  - 컨디션 점수(1~5점) 기반 자동 색상 매핑
    - 5~4점 → 🟢 / 3점 → 🟡 / 2~1점 → 🔴
- 날짜 선택 시 당일 이벤트 목록 조회
- 기록 유형별 아이콘 표시 (배변, 투약, 컨디션, 체중)

---

### 5-3. 이벤트 기반 기록 ⭐ 핵심

#### 입력 방식 (혼합)
- **FAB 버튼**: 홈 화면에서 오늘 날짜 기준 빠른 기록
- **캘린더**: 과거 날짜 선택 후 기록 추가 / 수정

#### 기록 유형

**💩 배변 기록**
- 배변 시간 기록
- 유형 선택: 소변 / 대변
- 상태 선택: 정상 / 묽음 / 딱딱함 / 혈변
- 초보자 배려: 상태 선택 시 각 상태의 의미와 주의사항 설명 팝업 제공

**🌡️ 컨디션 기록**
- 컨디션 점수 (1~5점 간단 체크)
- 증상 태그 선택 (구토, 기침, 무기력, 식욕부진 등 기본 태그 제공)
- 자유 메모 입력
- 사진 첨부 (1장)
- 초보자 배려: "이런 증상이 보이면 기록하세요" 예시 가이드 제공

**💊 투약 기록**
- 약 이름, 용량, 투여 방법, 투여 기간 등록
- 실제 투약 여부 체크
- 투약 완료 후 알림 설정 제안 (바텀시트)

**⚖️ 체중 기록**
- 체중 수치 입력
- 측정 방법 기록 (동물병원 / 가정용 체중계 / 안고 측정)
- 품종별 표준 체중 범위 비교 표시

---

### 5-4. 추세 그래프 (홈 카드) ⭐ 핵심

- 홈 화면 하단 카드 형태로 상시 노출
- 체중 변화 그래프 (최근 30일)
- 배변 횟수 주간 그래프
- 컨디션 점수 주간 그래프
- 카드 탭 시 상세 그래프 화면으로 이동

---

### 5-5. 예방접종 / 정기관리

- 접종 이력 등록 (종류, 날짜)
- 다음 접종 예정일 자동 계산 및 표시
- 심장사상충 / 구충 주기 관리
- 동물 종류별 기본 접종 스케줄 가이드 제공 (초보자용)
- 초보자 배려: "우리 아이에게 필요한 접종이 뭔가요?" 가이드 카드 제공

---

### 5-6. 알림 & 리마인더

> 시스템 자동 알림이 아닌, **사용자가 직접 원하는 시점을 지정**하는 방식

- 날짜 + 시간 직접 지정하여 알림 등록
- 반복 설정: 매일 / 매주 / N일마다
- 이벤트 입력 후 알림 설정 제안 (바텀시트 자동 표시)
- 알림 목록 조회 / 수정 / 삭제
- 알림 개수 무제한 (무료)

---

### 5-7. 응급 화면 ⭐ 핵심

- 상단 고정 버튼으로 원터치 접근
- 위치 기반 주변 동물병원 찾기
- 24시간 응급 동물병원 별도 표시
- 단골 병원 등록 및 빠른 전화 연결

---

### 5-8. 데이터 Export (유료)

- JSON key-value 포맷으로 전체 데이터 내보내기
- PDF 요약 리포트 내보내기
- 유료 기능 (₩2,900/회)

#### Export JSON 구조 원칙
- `export_version` 필드로 포맷 버전 관리
- `type` + `data` 구조로 새 기록 유형 추가 시 하위 호환 유지
- 구버전 앱이 신버전 export를 읽을 때 모르는 필드는 무시 처리
- ISO 8601 타임존 포함 날짜 포맷 사용
- 레코드 `id` 보존으로 중복 import 방지

#### Export JSON 예시
```json
{
  "export_version": "1.0",
  "exported_at": "2026-05-01T23:04:00+09:00",
  "app_version": "1.0.0",
  "pets": [
    {
      "id": "pet_001",
      "name": "콩이",
      "species": "dog",
      "breed": "말티즈",
      "birth_date": "2022-03-15",
      "gender": "male",
      "neutered": true,
      "records": [
        {
          "id": "rec_001",
          "type": "poop",
          "recorded_at": "2026-05-01T08:30:00+09:00",
          "data": {
            "poop_type": "normal",
            "urine": true
          }
        },
        {
          "id": "rec_002",
          "type": "condition",
          "recorded_at": "2026-05-01T09:00:00+09:00",
          "data": {
            "score": 4,
            "tags": ["활발함"],
            "memo": "오늘 산책 잘 함"
          }
        }
      ]
    }
  ]
}
```

## 7. UX 설계 원칙 (1차)

1. **온보딩 3단계**: 동물 등록 → 첫 기록 → 알림 설정
2. **FAB 빠른 기록**: 홈에서 탭 한 번으로 오늘 기록 즉시 접근
3. **캘린더 과거 기록**: 날짜 선택으로 과거 기록 추가 / 수정
4. **가이드 문구 내장**: 각 기록 화면에 "이렇게 활용하세요" 안내 상시 노출
5. **Empty State 친화적**: 기록이 없을 때 예시 화면 및 첫 기록 유도 CTA 제공
6. **툴팁 제공**: 입력 항목마다 초보자를 위한 설명 툴팁 제공
7. **알림 제안**: 이벤트 입력 완료 후 알림 설정 바텀시트 자동 표시
8. **다크모드 지원**: 기본 제공

## 8. 데이터 저장 정책 (1차)

| 항목 | 내용 |
|------|------|
| **저장 위치** | 기기 로컬 DB (SQLite 또는 동등 수준) |
| **클라우드 백업** | 미지원 (2차에서 유료 기능으로 추가 예정) |
| **데이터 복구** | 1차 미지원 — 앱 내 안내 문구 필수: "데이터는 이 기기에만 저장됩니다" |
| **Export 포맷** | JSON (key-value, 버전 명시, ISO 8601 날짜) |
| **2차 마이그레이션** | Export JSON을 마이그레이션 포맷으로 활용, 로컬 DB 스키마는 서버 스키마와 호환 가능하게 설계 |
